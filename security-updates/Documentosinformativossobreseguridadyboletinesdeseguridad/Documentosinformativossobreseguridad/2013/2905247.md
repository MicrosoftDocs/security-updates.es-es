Documento informativo sobre seguridad de Microsoft 2905247
==========================================================

La configuración de sitio ASP.NET no segura podría permitir la elevación de privilegios
---------------------------------------------------------------------------------------

Publicado: 10 de diciembre de 2013 | Actualizado: 9 de septiembre de 2014

**Versión:** 2.0

Información general
-------------------

### Resumen ejecutivo

Microsoft anuncia la disponibilidad de una actualización para Microsoft ASP.NET a fin de corregir una vulnerabilidad en el estado de vista de ASP.NET que se produce cuando la validación de código de autenticación de equipos (MAC) se ha deshabilitado mediante las opciones de configuración. La vulnerabilidad podría permitir la elevación de privilegios y afecta a Microsoft .NET Framework 1.1 Service Pack 1, Microsoft .NET Framework 2.0 Service Pack 2, Microsoft .NET Framework 3.5, Microsoft .NET Framework 3.5.1, Microsoft .NET Framework 4 y Microsoft .NET Framework 4.5/4.5.1.

Los sitios ASP.NET para los que se haya deshabilitado el MAC de estado de vista mediante las opciones de configuración son vulnerables al ataque. Un atacante que aprovechara la vulnerabilidad podría usar contenido HTTP especialmente diseñado para insertar código que se ejecute en el contexto de la cuenta de servicio en el servidor ASP.NET. Microsoft tiene constancia de que hay disponible información general que se podría usar para aprovechar esta vulnerabilidad, pero no tiene constancia de ningún ataque activo.

**Factores atenuantes**:

-   El MAC de estado de vista está habilitado de forma predeterminada en los sitios ASP.NET.

**Recomendación**. Microsoft recomienda que los clientes apliquen la acción sugerida para asegurarse de que el MAC de estado de vista de ASP.NET permanece habilitado en los sitios ASP.NET. Vea la sección **Acciones recomendadas** de este documento informativo para obtener más información.

**¿Cuál es el alcance de este documento informativo?**  
El propósito de este documento informativo es notificar a los clientes que Microsoft va a publicar una actualización para que los administradores puedan configurar sus servidores ASP.NET a fin de que puedan asegurarse de que el MAC de estado de vista permanece habilitado en todo momento, así como para proporcionar orientación general sobre cómo habilitar el MAC de estado de vista en los servidores IIS.
  
**¿Qué es el estado de vista?**    
El estado de vista es una característica de ASP.NET que permite a los desarrolladores mantener el estado de página y hacer persistentes los datos en un formulario web a través de las solicitudes POST, o las actualizaciones y cambios de página. El estado de vista lo usan con frecuencia los desarrolladores de ASP.NET y, por tanto, se encuentra en todas partes en los sitios ASP.NET. El estado de vista siempre se analiza, incluso si la propiedad EnableViewState se ha configurado en False. Para obtener más información, vea [Descripción del estado de vista de ASP.NET](http://msdn.microsoft.com/library/ms972976.aspx).
  
**¿Al deshabilitar el estado de vista se mitigará la vulnerabilidad?**  
No. El servidor ASP.NET siempre analiza el estado de vista, incluso si EnableViewState se ha configurado en False, independientemente de si la propiedad está configurada en web.config, en la directiva @Page o en una etiqueta ASP.NET. Un atacante puede insertar una propiedad de estado de vista en una publicación de cliente omitiendo la configuración EnableViewState.
  
**¿Qué es la validación de MAC de estado de vista?**  
La validación de MAC (código de autenticación de equipos) de estado de vista es una característica que provoca que ASP.NET genere un valor hash de los datos de estado de vista en el momento de generar la página. El valor hash se usa más adelante para la comparación con el estado de vista en una devolución posterior, lo que permite que el servidor compruebe si se ha manipulado el estado de vista. Esta tecnología garantiza que los datos de devolución no se han modificado de forma incorrecta y mitiga la vulnerabilidad descrita en este documento informativo.
  
**¿Para qué puede utilizar un atacante esta vulnerabilidad?**  
En la mayoría de los escenarios, un atacante que aprovechara esta vulnerabilidad podría elevar los privilegios al nivel de la cuenta de servicio que se ejecuta en el sitio ASP.NET vulnerable (con un MAC de estado de vista configurado incorrectamente).
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
Un atacante no autenticado podría enviar contenido HTTP especialmente diseñado al servidor atacado, lo que podría permitir al atacante ejecutar código en el servidor en el contexto de la cuenta de servicio que se ejecuta en el sitio ASP.NET.
  
**¿Cómo funciona esta actualización?**  
La actualización corrige la vulnerabilidad al provocar que el MAC de estado de vista esté habilitado en todo momento, lo que quita la posibilidad de deshabilitarlo en el servidor.
  
**¿Qué acciones adicionales deben realizar los clientes después de la instalación de la actualización?**  
La naturaleza de la revisión requiere que algunos clientes, especialmente los que usan ASP.NET en una granja de servidores web, lleven a cabo acciones adicionales para garantizar la disponibilidad continua de sus sitios ASP.NET. Vea en la sección **Acciones sugeridas** más adelante los pasos de configuración adicionales.
  
**¿Cómo determino la versión de Microsoft .NET Framework que está instalada?**    
Puede instalar y ejecutar múltiples versiones de .NET Framework en un sistema y puede instalar las versiones en cualquier orden. Hay varias formas de determinar las versiones de .NET Framework que están instaladas actualmente. Para obtener más información, vea el [artículo 318785 de Microsoft Knowledge Base](http://support.microsoft.com/kb/318785).
  
**¿Cuál es la diferencia entre .NET Framework 4.0 y .NET Framework 4.0 Client Profile?**    
Los paquetes redistribuibles de .NET Framework versión 4 están disponibles en dos perfiles: .NET Framework 4.0 y .NET Framework 4.0 Client Profile. .NET Framework 4.0 Client Profile es un subconjunto del perfil de .NET Framework 4.0 que está optimizado para aplicaciones cliente. Proporciona funcionalidad para la mayoría de las aplicaciones cliente, incluidas las características Windows Presentation Foundation (WPF), Windows Forms, Windows Communication Foundation (WCF) y ClickOnce. De este modo se habilita una implementación más rápida y un paquete de instalación de menor tamaño para las aplicaciones que se dirigen a .NET Framework 4.0 Client Profile. Para obtener más información, vea el artículo de MSDN [.NET Framework Client Profile](http://msdn.microsoft.com/en-us/library/cc656912.aspx).
  
**Tengo instalado .NET Framework 3.0 Service Pack 2; esta versión no está enumerada en el software afectado de este boletín. ¿Debo instalar una actualización?**    
Este boletín describe una vulnerabilidad que afecta a la capa de características de .NET Framework 2.0. El programa de instalación de .NET Framework 3.0 Service Pack 2 encadena la instalación de .NET Framework 2.0 Service Pack 2, por lo que la instalación del primero también instala el segundo. Por lo tanto, los clientes que tienen .NET Framework 3.0 Service Pack 2 instalado deben instalar las actualizaciones de seguridad de .NET Framework 2.0 Service Pack 2.
  
**Tengo .NET Framework 3.5 Service Pack 1 instalado. ¿Debo instalar alguna actualización?**  
Este boletín describe una vulnerabilidad que afecta a la capa de características de .NET Framework 2.0. El programa de instalación de .NET Framework 3.5 Service Pack 1 encadena la instalación de .NET Framework 2.0 Service Pack 2 y .NET Framework 3.0 Service Pack 2. Por lo tanto, los clientes que tienen .NET Framework 3.5 Service Pack 1 instalado deben instalar las actualizaciones de seguridad de .NET Framework 2.0 Service Pack 2.

Si tiene habilitadas y configuradas las actualizaciones automáticas para los productos de Microsoft, las actualizaciones se entregan cuando se publican, pero debe asegurarse de que están instaladas.
