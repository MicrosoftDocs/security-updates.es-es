Security Bulletin

Boletín de seguridad de Microsoft MS09-056 - Importante
=======================================================

Vulnerabilidades en Windows CryptoAPI podrían permitir la suplantación de identidad (spoofing) (974571)
-------------------------------------------------------------------------------------------------------

Publicado: martes, 13 de octubre de 2009 | Actualizado: miércoles, 14 de octubre de 2009

**Versión:** 1.0

### Información general

#### Resumen ejecutivo

Esta actualización de seguridad resuelve dos vulnerabilidades de las que se ha informado de forma privada en Microsoft Windows. Las vulnerabilidades podrían permitir la suplantación de identidad si un atacante obtiene acceso al certificado que usa el usuario final para la autenticación.

Esta actualización de seguridad se considera importante para todas las ediciones compatibles de Microsoft Windows 2000, Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008 y Windows 7. Para obtener más información, vea la subsección **Software afectado y no afectado**, en esta sección.

La actualización de seguridad corrige las vulnerabilidades al modificar CryptoAPI para rechazar los nombres de certificado que contienen terminadores nulos y para validar correctamente los identificadores de objeto ASN.1. Para obtener más información acerca de las vulnerabilidades, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad**.

**Recomendación.** La mayoría de los clientes tienen habilitada la actualización automática y no deben realizar ninguna acción porque esta actualización de seguridad se descargará e instalará automáticamente. Los clientes que no han habilitado la actualización automática deben buscar las actualizaciones e instalar esta actualización manualmente. Para obtener información sobre las opciones de configuración específicas de la actualización automática, vea el [artículo 294871 de Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para administradores e instalaciones empresariales, o usuarios finales que deseen instalar esta actualización de seguridad manualmente, Microsoft recomienda que los clientes apliquen la actualización a la primera oportunidad con el software de administración de actualizaciones o busquen las actualizaciones con el servicio [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Vea también la sección **Herramientas y consejos para la detección e implementación**, más adelante en este boletín.

**Problemas conocidos.** En el [artículo 974571 de Microsoft Knowledge Base](http://support.microsoft.com/kb/974571) se señalan los problemas detectados que pueden experimentar los clientes al instalar esta actualización de seguridad. Se documentan asimismo las soluciones propuestas. Cuando los problemas conocidos y las soluciones recomendadas corresponden sólo a versiones específicas de este software, este artículo proporciona vínculos a más artículos.

Vulnerabilidad de truncamiento nulo en nombre común de X.509 (CVE-2009-2510)  
----------------------------------------------------------------------------
  
<span></span>  
Existe una vulnerabilidad de suplantación de identidad (spoofing) en el componente CryptoAPI de Microsoft Windows al analizar la información ASN.1 de los certificados X.509. Un atacante que aprovechara esta vulnerabilidad podría suplantar a otro usuario o sistema.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2009-2510](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2510).
  
#### Factores atenuantes para la vulnerabilidad de truncamiento nulo en nombre común de X.509 (CVE-2009-2510)
  
Microsoft no ha identificado factores atenuantes para esta vulnerabilidad.
  
#### Soluciones provisionales para la vulnerabilidad de truncamiento nulo en nombre común de X.509 (CVE-2009-2510)
  
Microsoft no ha identificado soluciones provisionales para esta vulnerabilidad.
  
#### Preguntas más frecuentes sobre la vulnerabilidad de truncamiento nulo en nombre común de X.509 (CVE-2009-2510)
  
**¿Cuál es el alcance de esta vulnerabilidad?**    
Se trata de una vulnerabilidad de suplantación. Cuando se combina con otros ataques específicos, como la suplantación de DNS, puede permitir que el atacante consiga suplantar el certificado digital de un sitio web para cualquier aplicación que use CrytoAPI.
  
**¿Cuál es la causa de esta vulnerabilidad?**    
El componente CryptoAPI de Windows analiza incorrectamente un terminador nulo como el final de los valores identificados mediante un identificador de objeto (OID). Esto provoca que el componente CryptoAPI sólo analice parte del valor real. Cuando este valor parcial tiene significado para el usuario final, puede producir una situación en la que un atacante podría suplantar a otro usuario o sistema.
  
**¿Qué es el componente CryptoAPI de Windows?**    
El componente CryptoAPI proporciona servicios que permiten a los desarrolladores de aplicaciones agregar cifrado o descifrado de datos, autenticación mediante certificados digitales y codificación y descodificación a partir de ASN.1 (notación de sintaxis abstracta 1) a sus aplicaciones basadas en Windows. Los desarrolladores de aplicaciones pueden usar las funciones del componente CryptoAPI sin conocer los detalles de la implementación subyacente, de un modo muy parecido al uso que pueden hacer de una biblioteca de gráficos sin saber nada de la configuración de hardware de gráficos concreta. El componente CryptoAPI funciona con una serie de proveedores de servicios de cifrado que realizan las funciones de cifrado reales.
  
**¿Qué es ASN.1?**    
ASN.1 (notación de sintaxis abstracta 1) es un estándar de datos que usan numerosas aplicaciones y dispositivos del sector tecnológico para permitir la normalización y la comprensión de los datos en diferentes plataformas. ASN.1 no tiene una relación directa con ningún estándar, método de codificación, lenguaje de programación o plataforma de hardware en concreto. Se trata únicamente de un lenguaje para definir estándares. Los estándares están escritos en ASN.1.
  
Para obtener más información acerca de ASN.1, consulte el [artículo 252648 de Microsoft Knowledge Base](http://support.microsoft.com/kb/252648).
  
**¿Qué es el estándar X.509?**    
La especificación X.509 define un estándar para administrar las claves públicas a través de una infraestructura de claves públicas (PKI). Las claves públicas se mantienen en certificados X.509, que son documentos digitales que enlazan la declaración de identidad de un sujeto a una clave pública de un par de claves asimétricas públicas/privadas. Las declaraciones de identidad normalmente son comprensibles para las personas, como el nombre completo de una persona o su dirección de correo electrónico, un nombre de host de equipo o un nombre de dominio. Los certificados X.509 están respaldados y emitidos por un tercero de confianza, que se denomina entidad de certificación (CA).
  
**¿Que es un identificador de objeto u OID?**    
Los identificadores de objeto (OID) son valores numéricos emitidos por diferentes entidades para identificar de forma exclusiva elementos de datos, sintaxis y otras partes de aplicaciones distribuidas. Los OID se encuentran en aplicaciones OSI, directorios X.500, SNMP y otras aplicaciones en las que resulta importante la unicidad. Los OID se basan en una estructura de árbol, en la que una entidad de emisión superior, como ISO, asigna una rama del árbol a una entidad secundaria, que, a su vez, puede asignar subramas.
  
**¿Qué es un terminador nulo?**    
Un terminador nulo es un carácter con el valor cero. En determinados lenguajes de programación, como C, se usa como un carácter reservado para indicar el final de una cadena.
  
**¿Para qué puede utilizar un atacante esta vulnerabilidad?**    
Un atacante puede suplantar a otro usuario o sistema. Las aplicaciones que usan el componente CryptoAPI para analizar estos datos pueden devolver información al usuario que podría hacerle tomar decisiones de confianza incorrectas.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**    
Un atacante podría hacer un uso abusivo de esta vulnerabilidad al convencer a una entidad de certificación en la que confía el cliente para que firme un certificado que contiene un terminador nulo en el campo de nombre común (CN). A continuación, el atacante podría configurar un sitio web no autorizado que sirviera este certificado a un equipo de Windows. El equipo analizaría incorrectamente el campo del nombre común y ofrecería información de confianza incorrecta al usuario.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**    
Los clientes que exploran sitios web HTTPS o usan X.509 para los certificados, por ejemplo con el fin de proteger las conversaciones de correo electrónico, están más expuestos a esta vulnerabilidad.
  
**¿Cómo funciona esta actualización?**    
Esta actualización de seguridad corrige la vulnerabilidad al modificar el componente CryptoAPI para que rechace los nombres de certificado que contienen terminadores nulos.

Vulnerabilidad de desbordamiento de enteros en identificadores de objeto de X.509 (CVE-2009-2511)  
-------------------------------------------------------------------------------------------------
  
<span></span>

Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2009-2511](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2511).

#### Preguntas más frecuentes sobre la vulnerabilidad de desbordamiento de enteros en identificadores de objeto de X.509 (CVE-2009-2511)
  
**¿Cuál es el alcance de esta vulnerabilidad?**    
Se trata de una vulnerabilidad de suplantación. Cuando se combina con otros ataques específicos, como la suplantación de DNS, puede permitir que el atacante consiga suplantar el certificado digital de un sitio web para cualquier aplicación que use CrytoAPI.
  
**¿Cuál es la causa de esta vulnerabilidad?**    
Se produce un desbordamiento de enteros en el componente CryptoAPI de Windows al analizar los identificadores de objeto ASN.1 de los certificados X.509, como el nombre común (CN). Un atacante podría generar un certificado malintencionado que el componente CryptoAPI de Windows analizaría incorrectamente, lo que le permitiría suplantar a otro usuario o sistema.
  
**¿Qué es el componente CryptoAPI de Windows?**    
El componente CryptoAPI proporciona servicios que permiten a los desarrolladores de aplicaciones agregar cifrado o descifrado de datos, autenticación mediante certificados digitales y codificación y descodificación a partir de ASN.1 (notación de sintaxis abstracta 1) a sus aplicaciones basadas en Windows. Los desarrolladores de aplicaciones pueden usar las funciones del componente CryptoAPI sin conocer los detalles de la implementación subyacente, de un modo muy parecido al uso que pueden hacer de una biblioteca de gráficos sin saber nada de la configuración de hardware de gráficos concreta. El componente CryptoAPI funciona con una serie de proveedores de servicios de cifrado que realizan las funciones de cifrado reales.
  
**¿Qué es ASN.1?**    
ASN.1 (notación de sintaxis abstracta 1) es un estándar de datos que usan numerosas aplicaciones y dispositivos del sector tecnológico para permitir la normalización y la comprensión de los datos en diferentes plataformas. ASN.1 no tiene una relación directa con ningún estándar, método de codificación, lenguaje de programación o plataforma de hardware en concreto. Se trata únicamente de un lenguaje para definir estándares. Los estándares están escritos en ASN.1.
  
Para obtener más información acerca de ASN.1, consulte el [artículo 252648 de Microsoft Knowledge Base](http://support.microsoft.com/kb/252648).
  
**¿Qué es el estándar X.509?**    
La especificación X.509 define un estándar para administrar las claves públicas a través de una infraestructura de claves públicas (PKI). Las claves públicas se mantienen en certificados X.509, que son documentos digitales que enlazan la declaración de identidad de un sujeto a una clave pública de un par de claves asimétricas públicas/privadas. Las declaraciones de identidad normalmente son comprensibles para las personas, como el nombre completo de una persona o su dirección de correo electrónico, un nombre de host de equipo o un nombre de dominio. Los certificados X.509 están respaldados y emitidos por un tercero de confianza, que se denomina entidad de certificación (CA).
  
**¿Que es un identificador de objeto u OID?**    
Los identificadores de objeto (OID) son valores numéricos únicos emitidos por diferentes entidades para identificar de forma exclusiva elementos de datos, sintaxis y otras partes de aplicaciones distribuidas. Los OID se encuentran en aplicaciones OSI, directorios X.500, SNMP y otras aplicaciones en las que resulta importante la unicidad. Los OID se basan en una estructura de árbol, en la que una entidad de emisión superior, como ISO, asigna una rama del árbol a una entidad secundaria, que, a su vez, puede asignar subramas.
  
**¿Para qué puede utilizar un atacante esta vulnerabilidad?**    
Un atacante puede suplantar a otro usuario o sistema. Las aplicaciones que usan el componente CryptoAPI para analizar estos datos pueden devolver información al usuario que podría hacerle tomar decisiones de confianza incorrectas.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**    
Un atacante podría hacer un uso abusivo de esta vulnerabilidad al convencer a una entidad de certificación en la que confía el cliente para que firme un certificado que contiene un identificador de objeto con formato incorrecto. A continuación, el atacante podría configurar un sitio web no autorizado que sirviera este certificado a un cliente de Windows. Esta aplicación cliente analizaría incorrectamente el campo de nombre común y ofrecería información de confianza incorrecta al usuario.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**    
Las aplicaciones cliente que exploran sitios web HTTPS o usan X.509 para los certificados, por ejemplo con el fin de proteger las conversaciones de correo electrónico, están más expuestos a esta vulnerabilidad.
  
**¿Cómo funciona esta actualización?**    
Esta actualización de seguridad corrige la vulnerabilidad al provocar que el componente CryptoAPI valide incorrectamente los identificadores de objeto ASN.1.

Sí. Esa vulnerabilidad ya se ha divulgado. Se le ha asignado el número de vulnerabilidad y exposición común [CVE-2009-2511](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2511).

No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.
