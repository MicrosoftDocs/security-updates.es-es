Security Bulletin

Boletín de seguridad de Microsoft MS09-070 - Importante
=======================================================

Vulnerabilidades en Servicios de federación de Active Directory podrían permitir la ejecución remota de código (971726)
-----------------------------------------------------------------------------------------------------------------------

Publicado: martes, 8 de diciembre de 2009 | Actualizado: miércoles, 9 de diciembre de 2009

**Versión:** 1.1

### Información general

#### Resumen ejecutivo

Esta actualización de seguridad resuelve dos vulnerabilidades de las que se ha informado de forma privada en Microsoft Windows. La más grave de estas vulnerabilidades podría permitir la ejecución remota de código si un atacante envía una solicitud HTTP especialmente diseñada a un servidor web habilitado para ADFS. El atacante debe ser un usuario autenticado para aprovechar estas vulnerabilidades.

Esta actualización de seguridad se considera importante para las versiones afectadas de Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 y Windows Server 2008 x64 Edition. Para obtener más información, consulte la subsección **Software afectado y no afectado**, en esta sección.

La actualización de seguridad corrige las vulnerabilidades al modificar la validación que los servidores web habilitados para ADFS aplican a los encabezados de solicitud enviados por un cliente web. Para obtener más información acerca de las vulnerabilidades, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad**.

**Recomendación.**  La mayoría de los clientes tienen habilitada la actualización automática y no deben realizar ninguna acción porque esta actualización de seguridad se descargará e instalará automáticamente. Los clientes que no han habilitado la actualización automática deben buscar las actualizaciones e instalar esta actualización manualmente. Para obtener información sobre las opciones de configuración específicas de la actualización automática, vea el [artículo 294871 de Microsoft Knowledge Base](http://support.microsoft.com/kb/294871).

Para administradores e instalaciones empresariales, o usuarios finales que deseen instalar esta actualización de seguridad manualmente, Microsoft recomienda que los clientes apliquen la actualización a la primera oportunidad con el software de administración de actualizaciones o busquen las actualizaciones con el servicio [Microsoft Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=es-es).

Vea también la sección **Herramientas y consejos para la detección e implementación**, más adelante en este boletín.

**Problemas conocidos.** En el [artículo 971726 de Microsoft Knowledge Base](http://support.microsoft.com/kb/971726) se señalan los problemas detectados que pueden experimentar los clientes al instalar esta actualización de seguridad. Se documentan asimismo las soluciones propuestas.

**¿Qué es Servicios de federación de Active Directory?**  
Servicios de federación de Active Directory (ADFS) se basa en la reciente arquitectura de servicios web admitida por el sector, que está definida en las especificaciones WS-\*. ADFS facilita el uso del inicio de sesión único (SSO) para autenticar a los usuarios en múltiples aplicaciones web relacionadas durante el transcurso de una única sesión en línea. ADFS lleva a cabo esta tarea mediante el uso compartido de forma segura de la identidad y de los derechos a través de los límites de seguridad y empresariales.

**¿Cómo puedo comprobar si ADFS está instalado en mi servidor? **
Para comprobar que ADFS está instalado en el servidor, haga clic en **Inicio**, en **Panel de control**, en **Agregar o quitar programas**, haga doble clic en **Componentes de Windows** y, a continuación, busque **Servicios de federación** como componente instalado.

**¿Qué es Windows Server 2003 R2?**  
Windows Server 2003 R2 amplía el sistema operativo Windows Server 2003 al proporcionar una forma más eficaz de administrar y controlar el acceso a los recursos locales y remotos, a la vez que se integra fácilmente en el entorno de Windows Server 2003 existente. Windows Server 2003 R2 proporciona una plataforma web escalable y mejorada para la seguridad, interoperabilidad perfecta con los sistemas basados en UNIX y permite nuevos escenarios al incluir una administración simplificada de los servidores de bifurcación, mejora de la administración de las identidades y de los accesos, así como una administración más eficaz del almacenamiento. Windows Server 2003 R2 Datacenter Edition y Windows Server 2003 R2 Enterprise Edition también ofrecen un nuevo sistema de licencias dinámico que permite a los clientes obtener más valor de la virtualización de servidores.

Para obtener más información acerca de Windows Server 2003 R2, visite el sitio de [Windows Server 2003 R2](http://www.microsoft.com/spain/windowsserver2003/default.mspx).

**¿Qué son las especificaciones web WS-\*?**  

Son las especificaciones de una arquitectura de servicios web que se basa en estándares del sector, como, por ejemplo, SOAP (protocolo simple de acceso a objetos), XML, WSDL (Web Service Description Language) y UDDI (Universal Description, Discovery, and Integration). WS-\* proporciona la base para el suministro de soluciones empresariales completas e interoperables para la empresa extendida, incluida la capacidad de administrar la identidad y la seguridad federadas.

El modelo de servicios web se basa en la idea que de los sistemas empresariales están escritos en distintos lenguajes, con diferentes modelos de programación, que se ejecutan y se obtiene acceso a ellos desde numerosos tipos de dispositivos. Los servicios web constituyen un medio de crear sistemas distribuidos que pueden conectarse e interactuar entre sí de un modo sencillo y eficaz a través de Internet, independientemente del lenguaje en que estén escritos o la plataforma en que se ejecuten.

Vulnerabilidad de suplantación de identidad del inicio de sesión único en ADFS (CVE-2009-2508)  
----------------------------------------------------------------------------------------------
  
Una vulnerabilidad de suplantación de identidad en Servicios de federación de Active Directory podría permitir a un atacante suplantar a un usuario autenticado si el atacante tiene acceso a una estación de trabajo y un explorador web que haya usado recientemente el usuario atacado para obtener acceso a un sitio web que ofrece inicio de sesión único.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### Factores atenuantes para la vulnerabilidad de suplantación de identidad del inicio de sesión único en ADFS (CVE-2009-2508)
  
El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:
  
-   El atacante no puede elegir a quién puede suplantar. El atacante sólo puede suplantar al usuario cuyo token de autenticación haya podido obtener, por ejemplo, en un quiosco de acceso a Internet.  
-   El ataque sólo es posible antes de que se agote el tiempo de la sesión en el servidor. De forma predeterminada, las sesiones permanecen abiertas en el servidor durante 600 minutos. Si el atacante intenta reutilizar el token de autenticación de su víctima después de que haya transcurrido el periodo de tiempo de espera, no se podrá llevar a cabo el ataque.  
-   El atacante debe tener acceso al token de autenticación que ha usado el usuario anterior en dicho terminal. En el caso de un servidor habilitado para HTTPS, dicho token de autenticación estará protegido en el tránsito y el atacante deberá tener acceso al equipo de su víctima (por ejemplo, un quisco de Internet) para aprovechar la vulnerabilidad. En un quiosco de Internet, el usuario podría haber iniciado sesión en una aplicación de inicio de sesión único (SSO) de ADFS y, a continuación, salir de la aplicación. El atacante podría usar ese equipo para obtener acceso a la aplicación web como el usuario anterior, aunque éste haya cerrado la sesión.
  
#### Soluciones provisionales para la vulnerabilidad de suplantación de identidad del inicio de sesión único en ADFS (CVE-2009-2508)
  
Microsoft no ha identificado soluciones provisionales para esta vulnerabilidad.
  
#### Preguntas más frecuentes sobre la vulnerabilidad de suplantación de identidad del inicio de sesión único en ADFS (CVE-2009-2508)
  
**¿Cuál es el alcance de esta vulnerabilidad?**    
Se trata de una vulnerabilidad de suplantación. Un atacante que aprovechara esta vulnerabilidad podría suplantar a otro usuario, lo que permitiría al atacante obtener acceso como el usuario atacado a un sitio web que use la funcionalidad de inicio de sesión único de Servicios de federación de Active Directory (ADFS).
  
**¿Cuál es la causa de esta vulnerabilidad?**    
La vulnerabilidad se debe a que ADFS no realiza una validación suficiente de la administración de sesiones.
  
**¿Para qué puede usar un atacante esta vulnerabilidad?**    
En sistemas con acceso compartido, como los quioscos de Internet, un atacante que aprovechara esta vulnerabilidad podría obtener acceso a los recursos que pertenecen a otro usuario autenticado anteriormente. Este ataque sólo se podría ejecutar en el tiempo de espera de la sesión que haya configurado el administrador de ADFS, que, de forma predeterminada, es 600 minutos.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**    
El atacante podría aprovechar esta vulnerabilidad mediante el acceso a datos específicos desde la memoria caché del explorador y reutilizándolos para autenticarse en la aplicación web que implemente el inicio de sesión único (SSO) de ADFS. Después de que se agote el tiempo de espera de la sesión en el servidor, ADFS deshabilitará la sesión autenticada en el servidor y ya no permitirá que el atacante reutilice dichas credenciales.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**    
Los servidores que usan ADFS para el inicio de sesión único están afectados por esta vulnerabilidad. Esta vulnerabilidad no afecta directamente a los sistemas cliente.
  
**¿Cómo funciona esta actualización?**    
La actualización corrige la vulnerabilidad al descargar las credenciales de autenticación cuando el usuario cierra la sesión de la red.
  
**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**    
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable.
  
**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba usando esta vulnerabilidad?**    
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes.
  
Vulnerabilidad de ejecución remota de código en ADFS (CVE-2009-2509)  
--------------------------------------------------------------------
  
Existe una vulnerabilidad de ejecución remota de código en implementaciones de Servicios de federación de Active Directory. La vulnerabilidad se debe a una validación incorrecta de los encabezados de solicitud cuando un usuario autenticado se conecta a un servidor web habilitado para ADFS. Un atacante que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).
  
#### Factores atenuantes para la vulnerabilidad de ejecución remota de código en ADFS (CVE-2009-2509)
  
El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:
  
-   El atacante debe tener credenciales de inicio de sesión válidas en el servidor vulnerable. El ataque sólo se puede llevar a cabo después de autenticarse correctamente en el servidor que admite ADFS.
  
#### Soluciones provisionales para la vulnerabilidad de ejecución remota de código en ADFS (CVE-2009-2509)
  
Microsoft no ha identificado soluciones provisionales para esta vulnerabilidad.
  
#### Preguntas más frecuentes sobre la vulnerabilidad de ejecución remota de código en ADFS (CVE-2009-2509)
  
**¿Cuál es el alcance de esta vulnerabilidad?**    
Se trata de una vulnerabilidad de ejecución remota de código. Un atacante que aprovechara esta vulnerabilidad podría llevar a cabo acciones en el servidor IIS con los mismos derechos que la identidad de proceso de trabajo (WPI) que, de forma predeterminada, está configurada con privilegios de cuenta de servicio de red.
  
**¿Cuál es la causa de esta vulnerabilidad?**    
Esta vulnerabilidad se debe a que ADFS no valida correctamente los encabezados de solicitud enviados por los clientes autenticados.
  
**¿Para qué puede usar un atacante esta vulnerabilidad?**    
Un atacante que aprovechara esta vulnerabilidad podría llevar a cabo acciones en el servidor IIS con los mismos derechos que la identidad de proceso de trabajo (WPI). De forma predeterminada, la WPI está configurada con privilegios de cuenta de servicio de red. Los servidores IIS con páginas ASP cuyos grupos de aplicaciones estén configurados con una WPI que use una cuenta con privilegios administrativos pueden verse más gravemente afectados que los servidores IIS cuyo grupo de aplicaciones esté configurado con las opciones de WPI predeterminadas.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**    
Un atacante podría aprovechar la vulnerabilidad mediante el envío de una solicitud HTTP especialmente diseñada a un servidor web habilitado para ADFS. El atacante debe ser un usuario autenticado para aprovechar esta vulnerabilidad.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**    
Los servidores web habilitados para ADFS están expuestos a esta vulnerabilidad. Los sistemas cliente no están afectados por esta vulnerabilidad.
  
**¿Cómo funciona esta actualización?**    
Esta actualización corrige la validación que los servidores web habilitados para ADFS aplican a los encabezados de solicitud enviados por un cliente web.
  
**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**    
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable.

No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes.

**Comprobación de que se ha aplicado la actualización**

-   **Microsoft Baseline Security Analyzer**

    Para comprobar que una actualización de seguridad se ha aplicado en un sistema afectado, puede tratar de utilizar la herramienta Baseline Security Analyzer (MBSA) de Microsoft. Consulte la sección **Herramientas y consejos de detección e implementación**, anteriormente en este boletín, para obtener más información.

-   **Comprobación de la versión del archivo**

    Puesto que existen varias ediciones de Microsoft Windows, los pasos que se indican a continuación pueden ser diferentes en su sistema. En tal caso, consulte la documentación del producto para ejecutar los pasos necesarios.

    1.  Haga clic en **Inicio** y, después, en **Buscar**.
    2.  En el panel **Resultados de la búsqueda**, haga clic en **Todos los archivos y carpetas** del **Asistente para búsqueda**.
    3.  En el cuadro **Todo o parte del nombre de archivo**, escriba un nombre de archivo de la tabla de información de archivos correspondiente y, a continuación, haga clic en **Buscar**.
    4.  En la lista de archivos, haga clic con el botón secundario del mouse en un nombre de archivo de la tabla de información de archivos correspondiente y, después, haga clic en **Propiedades**.
        **Nota** Según la edición del sistema operativo o de los programas instalados, cabe la posibilidad de que algunos de los archivos enumerados en la tabla de información de archivos no estén instalados.
    5.  En la ficha **Versión**, determine la versión del archivo instalado en el sistema por comparación con la versión documentada en la tabla de información de archivos correspondiente.
        **Nota** Los atributos distintos a la versión del archivo pueden cambiar durante la instalación. La comparación de otros atributos de los archivos con los datos de la tabla de información de archivos no constituye un método apropiado para comprobar que se haya aplicado la actualización. Además, en ciertos casos, el nombre de los archivos se puede cambiar durante la instalación. Si no aparece información de la versión o del archivo, recurra a otro de los métodos disponibles para comprobar la instalación de la actualización.

-   **Comprobación de las claves del Registro**

    También puede comprobar los archivos que ha instalado esta actualización de seguridad si consulta las claves del Registro enumeradas en **Tabla de referencias** de esta sección.

    Estas claves del Registro pueden no contener una lista completa de archivos instalados. Además, estas claves del Registro pueden no crearse correctamente si el administrador o un OEM integran o incorporan esta actualización de seguridad en los archivos fuente de instalación de Windows.

#### Información sobre la implementación
  
**Instalación de la actualización**
  
Al instalar esta actualización de seguridad, el programa de instalación comprueba si uno o más de los archivos que se van a actualizar en el sistema se han actualizado anteriormente mediante una revisión de Microsoft.
  
Para obtener más información acerca de los términos que aparecen en este boletín, como *hotfix*, consulte el [artículo 824684 de Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).
  
Esta actualización de seguridad admite los parámetros de instalación que se indican a continuación.
  
| Parámetro         | Descripción                                                                                                                                               |  
|-------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Muestra ayuda sobre los parámetros admitidos.                                                                                                             |  
| **/quiet**        | Suprime la pantalla de estado o los mensajes de error.                                                                                                    |  
| **/norestart**    | Cuando se combina con **/quiet**, el sistema no se reiniciará después de la instalación, incluso si se necesita un reinicio para terminar la instalación. |
  
**Nota** Para obtener más información acerca del programa de instalación wusa.exe, consulte el [artículo 934307 de Microsoft Knowledge Base](http://support.microsoft.com/kb/934307).
  
**Comprobación de que se ha aplicado la actualización**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para comprobar que una actualización de seguridad se ha aplicado en un sistema afectado, puede tratar de utilizar la herramienta Baseline Security Analyzer (MBSA) de Microsoft. Consulte la sección **Herramientas y consejos de detección e implementación**, anteriormente en este boletín, para obtener más información.
  
-   **Comprobación de la versión del archivo**
  
    Puesto que existen varias ediciones de Microsoft Windows, los pasos que se indican a continuación pueden ser diferentes en su sistema. En tal caso, consulte la documentación del producto para ejecutar los pasos necesarios.
  
    1.  Haga clic en **Inicio** y, a continuación, escriba un nombre de archivo de actualización en **Iniciar búsqueda**.  
    2.  Cuando el archivo aparezca en **Programas**, haga clic con el botón secundario en el nombre de archivo y haga clic en **Propiedades**.  
    3.  En la ficha **General**, compare el tamaño de archivo con las tablas de información de archivo que se han proporcionado en el artículo de KB del boletín.  
    4.  También puede hacer clic en la ficha **Detalles** y comparar la información, como la versión de archivo y la fecha de modificación, con las tablas de información de archivo que se han proporcionado en el artículo de KB del boletín.  
    5.  Finalmente, también puede hacer clic en la ficha **Versiones anteriores** y comparar la información de archivo de la versión anterior del archivo con la información de archivo de la versión nueva o actualizada del archivo.
