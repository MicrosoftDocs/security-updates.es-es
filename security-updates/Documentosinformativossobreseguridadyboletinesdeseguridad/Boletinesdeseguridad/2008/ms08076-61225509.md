Security Bulletin

Boletín de seguridad de Microsoft MS08-076 - Importante
=======================================================

Vulnerabilidades en los componentes de Windows Media podrían permitir la ejecución remota de código (959807)
------------------------------------------------------------------------------------------------------------

Publicado: martes, 9 de diciembre de 2008 | Actualizado: martes, 24 de noviembre de 2009

**Versión:** 5.0

### Información general

#### Resumen ejecutivo

Esta actualización de seguridad resuelve dos vulnerabilidades de las que se ha informado de forma privada en los siguientes componentes de Windows Media: Reproductor de Windows Media, módulo de tiempo de ejecución de Windows Media Format y Servicios de Windows Media. La vulnerabilidad más grave podría permitir la ejecución de código remoto. Si un usuario inicia sesión con derechos de usuario administrativos, un intruso que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.

Esta actualización de seguridad se considera importante para Reproductor de Windows Media 6.4, módulo de tiempo de ejecución de Windows Media Format 7.1, módulo de tiempo de ejecución de Windows Media Format 9.0, módulo de tiempo de ejecución de Windows Media Format 9.5, módulo de tiempo de ejecución de Windows Media Format 11, Servicios de Windows Media 4.1, Servicios de Windows Media 9 Series y Servicios de Windows Media 2008. Para obtener más información, vea la subsección, **Software afectado y no afectado**, en esta sección.

La actualización de seguridad corrige la vulnerabilidad al modificar la manera en que se validan las respuestas de autenticación de Windows Media. La actualización de seguridad corrige la segunda vulnerabilidad al garantizar que los clientes de Windows Media tratan los servidores que usan direcciones ISATAP como externos. Para obtener más información acerca de las vulnerabilidades, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad**.

**Recomendación.** Microsoft recomienda que los clientes instalen la actualización lo antes posible.

**Problemas conocidos.** Ninguna

#### Software afectado y no afectado

El software siguiente se ha probado para determinar las versiones o ediciones que están afectadas. Otras versiones o ediciones han llegado al final de su ciclo de vida o no están afectadas. Para determinar el ciclo de vida del soporte técnico de su versión o edición de software, visite [Ciclo de vida del soporte técnico de Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Preguntas más frecuentes (P+F) relacionadas con esta actualización de seguridad
-------------------------------------------------------------------------------

<span></span>
**¿Por qué se ha vuelto a publicar este boletín el 24 de noviembre de 2009?**  
Microsoft ha vuelto a publicar este boletín para ofrecer un nuevo paquete de actualización específico de los sistemas Windows XP Embedded para corregir los problemas de detección e instalación de los paquetes de actualización de Windows XP originales incluidos con este boletín.

Además, Microsoft revisó los paquetes de actualización existentes para Windows XP Service Pack 2 (KB952069) y Windows XP Service Pack 3 (KB952069) a fin de evitar que los paquetes se ofrezcan a los sistemas Windows XP Embedded. Los clientes que ya han instalado esta actualización de seguridad están protegidos y no deben realizar ninguna acción.

Los clientes que usan el sistema operativo Windows XP Embedded deben aplicar esta actualización lo antes posible. Los clientes también pueden descargar manualmente la actualización para [Windows XP Embedded Service Pack 2](http://www.microsoft.com/downloads/details.aspx?familyid=c8f59b79-7d09-4976-a678-9aef5975c186) (KB972187) o la actualización para [Windows XP Embedded Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=502dcf76-aec3-48bf-930d-0c02b1604cd1) (KB972187). Ninguna otra actualización está afectada por esta nueva publicación.

**¿Qué es el sistema operativo Windows XP Embedded?**  
Microsoft Windows XP Embedded es una versión formada por componentes del sistema operativo Windows XP Professional que aporta el conjunto de características enriquecidas de Windows XP Professional a los dispositivos incrustados. Windows XP Embedded está optimizado para el sector minorista, con capacidades de red integradas y compatibilidad plug and play basada en estándares. Windows XP Embedded permite la conexión de varios periféricos de dispositivos del sector minorista u hotelero en un único sistema de punto de servicio (POS). Para obtener más información acerca de los sistemas operativos Windows XP Embedded, visite el [Centro para desarrolladores de Microsoft Windows Embedded](http://msdn.microsoft.com/en-us/windowsembedded/default.aspx).

**¿Por qué se ha vuelto a publicar este boletín el 13 de enero de 2009?**  
Microsoft ha vuelto a publicar este boletín para ofrecer nuevos paquetes de actualización para Módulo de tiempo de ejecución de Windows Media Format 9.5 en Windows XP Service Pack 2 (KB952069) y Windows XP Service Pack 3 (KB952069). Los nuevos paquetes de actualización corrigen un problema que provoca una instalación incompleta en los sistemas Windows XP que ejecutan Módulo de tiempo de ejecución de Windows Media Format 9.5 en los que se ha instalado KB944110, una actualización independiente, antes de instalar KB952069. Estos clientes siguen siendo vulnerables y deben instalar los paquetes de actualización que se han vuelto a publicar. Ningún otro paquete de actualización está afectado por esta nueva publicación. Los clientes que ejecutan todas las demás versiones admitidas y afectadas de Reproductor de Windows Media, Módulo de tiempo de ejecución de Windows Media Format y Servicios de Windows Media que ya hayan aplicado correctamente los paquetes de actualización de seguridad originales no tienen que realizar ninguna acción. Los paquetes de actualización de seguridad que se han vuelto a publicar están disponibles en los mismos canales de distribución que la actualización de seguridad original, incluyendo Actualizaciones automáticas, Windows Update y Windows Server Update Services.

Microsoft también ha revisado este boletín para incluir Reproductor de Windows Media 6.4 y Servicios de Windows Media 4.1 como afectados en todas las ediciones de Microsoft Windows 2000 Service Pack 4 (no sólo Microsoft Windows 2000 Server Service Pack 4). Se aplica la misma actualización; no hay nuevos paquetes de actualización para este sistema operativo. No obstante, los clientes a los que se les ha ofrecido esta actualización (KB954600 para Reproductor de Windows Media 6.4 o KB952068 para Servicios de Windows Media 4.1) pero no la han aplicado tienen que hacerlo.

**¿Dónde están los detalles de información de archivos?**  
Los detalles de información de archivos se pueden encontrar en el [artículo 959807 de Microsoft Knowledge Base](http://support.microsoft.com/kb/959807).

**¿Por qué soluciona esta actualización varios problemas de vulnerabilidad de seguridad detectados?**  
Esta actualización aborda diversos puntos vulnerables porque las modificaciones necesarias para solucionar estos aspectos están ubicadas en archivos relacionados. En lugar de instalar varias actualizaciones que son prácticamente iguales, los clientes sólo necesitan instalar esta actualización.

**Uso una versión anterior del software tratado en este boletín de seguridad. ¿Qué debo hacer?**  
El software afectado que se enumera en este boletín se ha probado para determinar las versiones que están afectadas. Otras versiones o ediciones han pasado su ciclo de vida del soporte técnico. Para determinar el ciclo de vida del soporte técnico de su versión de software, visite [Ciclo de vida del soporte técnico de Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Los clientes que tengan esas versiones o ediciones anteriores del software deberían plantearse la migración a versiones con soporte técnico para evitar la exposición a vulnerabilidades. Vulnerabilidad de SPN (CVE-2008-3009)
-------------------------------------

<span></span>
Existe una vulnerabilidad de reflexión de credenciales en los componentes de Windows Media que podría permitir a un atacante ejecutar código con los mismos derechos que el usuario local o con las credenciales de distribución de Servicios de Windows Media. La vulnerabilidad se debe a puntos débiles en las implementaciones del nombre principal de servicio (SPN) en los componentes de Windows Media.

Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2008-3009](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3009).

#### Factores atenuantes para la vulnerabilidad de SPN (CVE-2008-3009)

El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:

-   Servicios de Windows Media es un componente en todas las plataformas de Windows Server y no se instala de forma predeterminada.
-   En el caso de un ataque basado en Web, el atacante tendría que alojar un sitio web que contuviera una página web para aprovechar esta vulnerabilidad. El atacante no podría obligar a los usuarios a visitar un sitio Web malintencionado. Por tanto, tendría que atraerlos al sitio web, normalmente incitándoles a hacer clic en un vínculo que le lleve al mismo. Al hacer clic en el vínculo, se les pedirá que realicen varias acciones. El ataque solamente podría tener lugar después de realizar estas acciones.
-   Para Windows Server 2008 y Windows Server 2008 para sistemas x64, módulo de tiempo de ejecución de Windows Media Format 11 sólo está en el sistema cuando se instala Experiencia de escritorio.

#### Soluciones provisionales para la vulnerabilidad de SPN (CVE-2008-3009)

Microsoft no ha identificado soluciones provisionales para esta vulnerabilidad.

#### Preguntas más frecuentes sobre la vulnerabilidad de SPN (CVE-2008-3009)

**¿Cuál es el alcance de esta vulnerabilidad?**  
Se trata de una vulnerabilidad de ejecución remota de código. Un atacante que aprovechara esta vulnerabilidad podría ejecutar código con los mismos privilegios que el usuario que ha iniciado la sesión. Si un usuario inicia sesión con derechos de usuario administrativos, un intruso que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.  Un atacante que aprovechara esta vulnerabilidad también podría ejecutar código con las credenciales de distribución de Servicios de Windows Media. En este caso, un atacante podría ejecutar código con los mismos derechos que la cuenta SERVICIO DE RED.

**¿Cuál es la causa de esta vulnerabilidad?**  
Los componentes de Windows Media no aceptan correctamente las protecciones de reflexión de credenciales NTLM para garantizar que las credenciales de un usuario no se reflejan en el sistema del usuario y se usan para suplantarlo.

**¿Qué son los nombres principales de servicio (SPN)?**  
Un nombre principal de servicio (SPN) es el nombre por el que un cliente identifica de forma exclusiva una instancia de un servicio. El SPN proporciona un tipo de identidad de servicio que se usa para autenticar el servicio ante el cliente. Cuando un cliente desea conectarse a un servicio, busca una instancia de él, elabora un SPN para dicha instancia, se conecta al servicio y presenta el SPN para que lo autentique el servicio. Para obtener información adicional acerca de SPN, vea [Identidad de servicio y autenticación](http://msdn.microsoft.com/en-us/library/ms733130.aspx). Para obtener información adicional acerca del formato que deben tener los SPN, vea [Formatos de nombre para SNP únicos](http://msdn.microsoft.com/en-us/library/ms677601(vs.85).aspx).

**¿Para qué puede usar un atacante esta vulnerabilidad?**  
Un intruso que aprovechara esta vulnerabilidad podría conseguir el mismo nivel de derechos de usuario que el usuario que ha iniciado sesión. Si un usuario inicia sesión con derechos de usuario administrativos, un intruso que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.

**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
Esta vulnerabilidad requiere que un usuario con una versión afectada de los componentes de Windows Media tenga acceso a un servidor malintencionado. Un atacante podría hospedar un recurso compartido de servidor malintencionado o un sitio web que requiera autenticación. Cuando el cliente se conecta y se autentica en el servidor malintencionado, el atacante puede usar las credenciales que proporciona el cliente para reflejarse en él o reenviarlas a otros servidores e imitarlo. El atacante no podría obligar a los usuarios a visitar un recurso compartido de servidor o un sitio web especialmente diseñados. Por lo tanto, tendría que atraerlos al recurso compartido de servidor o al sitio web; por lo general, convenciéndoles para que hagan clic en un vínculo de un mensaje de correo electrónico o de Instant Messenger que lleve a los usuarios al sitio del atacante.

**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Cualquier sistema que use las versiones afectadas de Reproductor de Windows Media, módulo de tiempo de ejecución de Windows Media Format o Servicios de Windows Media está expuesto a esta vulnerabilidad. Cualquier sistema en el que se usen estos componentes de forma habitual, como estaciones de trabajo y servidores de Terminal Server, está expuesto a esta vulnerabilidad.

**¿Cómo funciona esta actualización?**  
La actualización de seguridad corrige la vulnerabilidad al modificar la manera en que se validan las respuestas de autenticación de Componentes de Windows Media para evitar la repetición de credenciales.

**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable. En el momento de emitir este boletín de seguridad, Microsoft no había recibido información alguna que indicara que esta vulnerabilidad se hubiera divulgado.

**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba usando esta vulnerabilidad?**  
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.

Vulnerabilidad de ISATAP (CVE-2008-3010)
----------------------------------------

<span></span>
Existe una vulnerabilidad de divulgación de información en versiones compatibles de los componentes de Windows Media que podría conllevar la divulgación de credenciales NTLM. Cuando componente de Windows Media que obtenga acceso a una dirección URL que use una dirección ISATAP puede enviar las credenciales NTLM del usuario al servidor que hospeda la dirección URL. Esto podría permitir a un atacante externo a la zona de intranet recopilar credenciales NTLM para un entorno empresarial.

Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2008-3010](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3010).

#### Factores atenuantes para la vulnerabilidad de ISATAP (CVE-2008-3010)

El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:

-   Si se siguen los procedimientos recomendados relativos al uso de firewalls y se implementan las configuraciones de servidores de seguridad predeterminadas estándar, puede contribuirse a proteger recursos de red de los ataques que se originen fuera del ámbito de la empresa. Se recomienda que los sistemas conectados a Internet tengan expuesta la cantidad mínima de puertos y protocolos.
-   En el caso de un ataque basado en Web, el atacante tendría que alojar un sitio web que contuviera una página web para aprovechar esta vulnerabilidad. El atacante no podría obligar a los usuarios a visitar un sitio Web malintencionado. Por tanto, tendría que atraerlos al sitio web, normalmente incitándoles a hacer clic en un vínculo que le lleve al mismo. Al hacer clic en el vínculo, se les pedirá que realicen varias acciones. El ataque solamente podría tener lugar después de realizar estas acciones.
-   Para Windows Server 2008 y Windows Server 2008 para sistemas x64, módulo de tiempo de ejecución de Windows Media Format 11 sólo está en el sistema cuando se instala Experiencia de escritorio.

#### Soluciones provisionales para la vulnerabilidad de ISATAP (CVE-2008-3010)

La solución provisional hace referencia a un cambio de opción o configuración que no corrige la vulnerabilidad subyacente pero ayuda a bloquear los tipos de ataque conocidos antes de aplicar la actualización. Microsoft ha probado las soluciones provisionales y señala en la explicación si una solución provisional reduce la funcionalidad

-   **Bloquear el tipo de protocolo de Internet 41 (ISATAP) en el firewall**

    El protocolo ISATAP está definido como tipo de protocolo 41 en [RFC4214: Protocolo de direccionamiento de túnel automático en el sitio (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt). Al bloquear este tipo de protocolo en el firewall, contribuirá a proteger los sistemas situados detrás del firewall frente a los intentos de aprovechar esta vulnerabilidad. Microsoft recomienda bloquear toda la comunicación entrante no solicitada de Internet para ayudar a impedir ataques que puedan utilizar otros puertos o protocolos.

#### Preguntas más frecuentes sobre la vulnerabilidad de ISATAP (CVE-2008-3010)

**¿Cuál es el alcance de esta vulnerabilidad?**  
Esta vulnerabilidad afecta a la divulgación de información. Un atacante que aprovechara esta vulnerabilidad podría obtener las credenciales NTLM del usuario que ha iniciado la sesión. Estas credenciales se pueden reflejar posteriormente en el sistema del usuario para ejecutar código con los mismos derechos que el usuario que ha iniciado sesión.

**¿Cuál es la causa de esta vulnerabilidad?**  
Los componentes de Windows Media afectados tratan incorrectamente los recursos de Internet como parte de una zona de intranet local al conectarse a un servidor que usa una dirección ISATAP.

**¿Qué es ISATAP?**  
El protocolo de direccionamiento de túnel automático en el sitio (ISATAP) proporciona conectividad IPv6 en una intranet IPv4. El RFC para ISATAP lo hospeda Internet Engineering Task Force (IETF). Para obtener información acerca de ISATAP, vea [RFC4214: Protocolo de direccionamiento de túnel automático en el sitio (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt).

**¿Qué es NTLM?**  
NTLM es un protocolo de autenticación que se basa en un mecanismo de desafío/respuesta.

**¿Para qué puede usar un atacante esta vulnerabilidad?**  
Un atacante que aprovechara esta vulnerabilidad podría obtener las credenciales NTLM del usuario.  Estas credenciales se pueden reflejar posteriormente en el sistema del usuario para ejecutar código con los mismos derechos que el usuario que ha iniciado sesión.

**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
Esta vulnerabilidad requiere que un usuario con una versión afectada de los componentes de Windows Media tenga acceso a un servidor malintencionado. Un atacante podría hospedar un recurso compartido de servidor malintencionado o un sitio web que requiera autenticación. Cuando el cliente se conecta y se autentica en el servidor malintencionado, el atacante puede usar las credenciales que proporciona el cliente para reflejarse en él o reenviarlas a otros servidores e imitarlo. El atacante no podría obligar a los usuarios a visitar un recurso compartido de servidor o un sitio web especialmente diseñados. Por lo tanto, tendría que atraerlos al recurso compartido de servidor o al sitio web; por lo general, convenciéndoles para que hagan clic en un vínculo de un mensaje de correo electrónico o de Instant Messenger que lleve a los usuarios al sitio del atacante.

**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Cualquier sistema que use las versiones afectadas de Reproductor de Windows Media, módulo de tiempo de ejecución de Windows Media Format o Servicios de Windows Media está expuesto a esta vulnerabilidad. **¿Cómo funciona esta actualización?**  
La actualización elimina la vulnerabilidad al garantizar que los clientes de Componentes de Windows Media tratan a los servidores que usan direcciones ISATAP como externos y no transmiten las credenciales del usuario.

**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable. En el momento de emitir este boletín de seguridad, Microsoft no había recibido información alguna que indicara que esta vulnerabilidad se hubiera divulgado.

**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba usando esta vulnerabilidad?**  
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.

-   **Comprobación de la versión del archivo**

    Puesto que existen varias ediciones de Microsoft Windows, los pasos que se indican a continuación pueden ser diferentes en su sistema. En tal caso, consulte la documentación del producto para ejecutar los pasos necesarios.

    1.  Haga clic en **Inicio** y, después, en **Buscar**.
    2.  En el panel **Resultados de la búsqueda**, haga clic en **Todos los archivos y carpetas** del **Asistente para búsqueda**.
    3.  En el cuadro **Todo o parte del nombre de archivo**, escriba un nombre de archivo de la tabla de información de archivos correspondiente y, a continuación, haga clic en **Buscar**.
    4.  En la lista de archivos, haga clic con el botón secundario del mouse en un nombre de archivo de la tabla de información de archivos correspondiente y, después, haga clic en **Propiedades**.
        **Nota** Según la edición del sistema operativo o de los programas instalados, cabe la posibilidad de que algunos de los archivos enumerados en la tabla de información de archivos no estén instalados.
    5.  En la ficha **Versión**, determine la versión del archivo instalado en el sistema por comparación con la versión documentada en la tabla de información de archivos correspondiente.
        **Nota** Los atributos distintos a la versión del archivo pueden cambiar durante la instalación. La comparación de otros atributos de los archivos con los datos de la tabla de información de archivos no constituye un método apropiado para comprobar que se haya aplicado la actualización. Además, en ciertos casos, el nombre de los archivos se puede cambiar durante la instalación. Si no aparece información de la versión o del archivo, recurra a otro de los métodos disponibles para comprobar la instalación de la actualización.

#### Información sobre la implementación

**Instalación de la actualización**

Al instalar esta actualización de seguridad, el programa de instalación comprueba si uno o más de los archivos que se van a actualizar en el sistema se han actualizado anteriormente mediante una revisión de Microsoft.

Si ha instalado anteriormente una revisión para actualizar alguno de estos archivos, el programa de instalación copia los archivos SP2QFE o SP3QFE en el sistema. De lo contrario, el programa de instalación copia los archivos SP2GDR o SP3GDR en el sistema. Las actualizaciones de seguridad pueden no incluir todas las variaciones de estos archivos. Para obtener más información acerca de este comportamiento, consulte el [artículo 824994 de Microsoft Knowledge Base](http://support.microsoft.com/kb/824994).

Para obtener más información acerca del programa de instalación, visite el [sitio web de Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para obtener más información acerca de los términos que aparecen en este boletín, como *hotfix*, consulte el [artículo 824684 de Microsoft Knowledge Base](http://support.microsoft.com/kb/824684).

**Comprobación de que se ha aplicado la actualización**

-   **Microsoft Baseline Security Analyzer**

    Para comprobar que una actualización de seguridad se ha aplicado en un sistema afectado, puede tratar de utilizar la herramienta Baseline Security Analyzer (MBSA) de Microsoft. Consulte la sección **Herramientas y consejos de detección e implementación**, anteriormente en este boletín, para obtener más información.

-   **Comprobación de la versión del archivo**

    Puesto que existen varias ediciones de Microsoft Windows, los pasos que se indican a continuación pueden ser diferentes en su sistema. En tal caso, consulte la documentación del producto para ejecutar los pasos necesarios.

    1.  Haga clic en **Inicio** y, después, en **Buscar**.
    2.  En el panel **Resultados de la búsqueda**, haga clic en **Todos los archivos y carpetas** del **Asistente para búsqueda**.
    3.  En el cuadro **Todo o parte del nombre de archivo**, escriba un nombre de archivo de la tabla de información de archivos correspondiente y, a continuación, haga clic en **Buscar**.
    4.  En la lista de archivos, haga clic con el botón secundario del mouse en un nombre de archivo de la tabla de información de archivos correspondiente y, después, haga clic en **Propiedades**.
        **Nota** Según la edición del sistema operativo o de los programas instalados, cabe la posibilidad de que algunos de los archivos enumerados en la tabla de información de archivos no estén instalados.
    5.  En la ficha **Versión**, determine la versión del archivo instalado en el sistema por comparación con la versión documentada en la tabla de información de archivos correspondiente.
        **Nota** Los atributos distintos a la versión del archivo pueden cambiar durante la instalación. La comparación de otros atributos de los archivos con los datos de la tabla de información de archivos no constituye un método apropiado para comprobar que se haya aplicado la actualización. Además, en ciertos casos, el nombre de los archivos se puede cambiar durante la instalación. Si no aparece información de la versión o del archivo, recurra a otro de los métodos disponibles para comprobar la instalación de la actualización.

**Instalación de la actualización**

Al instalar esta actualización de seguridad, el programa de instalación comprueba si uno o más de los archivos que se van a actualizar en el equipo han sido actualizados anteriormente mediante una revisión de Microsoft.

Si ya había instalado una revisión para actualizar uno de estos archivos, el programa de instalación copia los archivos SP1QFE o SP2QFE en el sistema. De lo contrario, el programa de instalación copia los archivos SP1GDR o SP2GDR en el sistema. Las actualizaciones de seguridad pueden no incluir todas las variaciones de estos archivos. Para obtener más información acerca de este comportamiento, consulte el [artículo 824994 de Microsoft Knowledge Base](http://support.microsoft.com/kb/824994).

Para obtener más información acerca del programa de instalación, visite el [sitio web de Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).
