Security Bulletin

Boletín de seguridad de Microsoft MS08-047 - Importante
=======================================================

Una vulnerabilidad en el procesamiento de la directiva IPsec podría permitir la divulgación de información (953733)
-------------------------------------------------------------------------------------------------------------------

Publicado: martes, 12 de agosto de 2008 | Actualizado: miércoles, 13 de agosto de 2008

**Versión:** 1.1

### Información general

#### Resumen ejecutivo

Esta actualización resuelve una vulnerabilidad de la que se ha informado de forma privada en la manera en que se aplican determinadas reglas de seguridad del protocolo de Internet de Windows (IPsec). Esta vulnerabilidad podría provocar que determinados sistemas ignoren las directivas IPsec y transmitan el tráfico de red en texto sin cifrar. De este modo se divulgaría información destinada a estar cifrada en la red. Un atacante que vea el tráfico en la red podría ver y modificar el contenido del tráfico. Observe que esta vulnerabilidad no permite a los atacantes ejecutar código o aumentar sus derechos de usuario directamente, pero podría servir para recopilar información útil que pudiera usarse para tratar de sacar más provecho del sistema afectado o la red afectados.

Esta actualización de seguridad se considera importante para todas las versiones compatibles de Windows Vista y Windows Server 2008. Para obtener más información, consulte la subsección **Software afectado y no afectado**, en esta sección**.**

La actualización de seguridad corrige la vulnerabilidad al garantizar que las reglas IPsec se procesan correctamente. Para obtener más información acerca de la vulnerabilidad, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad**.

**Recomendación.** Microsoft recomienda que los clientes instalen la actualización lo antes posible.

**Problemas conocidos.** Ninguna

Vulnerabilidad de divulgación de información en directiva IPsec (CVE-2008-2246)  
-------------------------------------------------------------------------------
  
<span></span>
Existe una vulnerabilidad de divulgación de información en la manera en que las directivas IPsec se importan en los dominios de Windows Server 2008 desde los dominios de Windows Server 2003. Esta vulnerabilidad podría provocar que determinados sistemas ignoren las directivas IPsec y transmitan el tráfico de red en texto sin cifrar. De este modo se podría divulgar información destinada a estar cifrada en la red. Un atacante que intercepte el tráfico en la red podría ver y modificar el contenido del tráfico. Observe que esta vulnerabilidad no permite a los atacantes ejecutar código o aumentar sus derechos de usuario directamente, pero podría servir para recopilar información útil que pudiera usarse para tratar de sacar más provecho del sistema afectado.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2008-2246](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2246).
  
#### Factores atenuantes para la vulnerabilidad de divulgación de información en directiva IPsec (CVE-2008-2246)
  
El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:
  
-   En Windows Vista y Windows Server 2008, la regla de respuesta predeterminada no está habilitada automáticamente. La regla de respuesta predeterminada sólo se habilita automáticamente cuando se crea una directiva IPsec mediante las versiones de Windows anteriores a Windows Vista y Windows Server 2008.
  
#### Soluciones provisionales para la vulnerabilidad de divulgación de información en directiva IPsec (CVE-2008-2246)
  
La solución provisional hace referencia a un cambio de opción o configuración que no corrige la vulnerabilidad subyacente pero ayuda a bloquear los tipos de ataque conocidos antes de aplicar la actualización. Microsoft ha probado las soluciones provisionales y señala en la explicación si una solución provisional reduce la funcionalidad
  
-   No seleccione "Regla de respuesta predeterminada" durante la creación de la directiva IPsec o desactive esta regla de las directivas existentes ya que esta regla ya no es válida en Windows Vista y Windows Server 2008, y sólo es aplicable a versiones anteriores de Windows. Para emular esta regla en Windows Vista y Windows Server 2008, consulte el [artículo 942964 de Microsoft Knowledge Base](http://support.microsoft.com/kb/942964/).
  
#### Preguntas más frecuentes sobre la vulnerabilidad de divulgación de información en directiva IPsec (CVE-2008-2246)
  
**¿Cuál es el alcance de esta vulnerabilidad?**    
Esta vulnerabilidad afecta a la divulgación de información. Un atacante que aprovechara esta vulnerabilidad podría leer el contenido del tráfico de red que, de lo contrario, estaría cifrado. Observe que esta vulnerabilidad no permite a los atacantes ejecutar código o aumentar sus derechos de usuario directamente, pero podría servir para recopilar información útil que pudiera usarse para tratar de sacar más provecho del sistema afectado.
  
**¿Cuál es la causa de esta vulnerabilidad?**    
Esta vulnerabilidad se debe a un error cuando la directiva IPsec predeterminada se importa desde un dominio de Windows Server 2003 a un dominio de Windows Server 2008. En determinadas circunstancias, este error podría provocar que se ignoraran todas las reglas de IPsec.
  
**¿Qué es IPsec?**    
El protocolo de seguridad de protocolo de Internet (IPsec) es un marco de estándares abiertos para garantizar las comunicaciones privadas y seguras a través de redes IP (Protocolo de Internet) mediante el uso de servicios de seguridad criptográficos. El grupo de trabajo de IPsec Internet Engineering Task Force (IETF) define los estándares de IPsec.
  
IPsec es la dirección a largo plazo de las redes seguras. Proporciona protección agresiva contra los ataques a las redes privadas y de Internet mediante seguridad completa. Los únicos equipos que deben conocer la protección IPsec son el emisor y el receptor de la comunicación. IPsec proporciona la capacidad de proteger la comunicación entre grupos de trabajo, equipos de red de área local, clientes y servidores de dominio, sucursales (que pueden estar alejadas físicamente), extranets y clientes itinerantes. Para obtener más información acerca de IPsec, visite el siguiente [sitio web de Microsoft](http://www.microsoft.com/technet/network/ipsec/ipsecfaq.mspx).
  
**¿Para qué puede usar un atacante esta vulnerabilidad?**    
Un atacante que aproveche esta vulnerabilidad puede ver y, en algunos casos, modificar el tráfico de red en texto sin cifrar destinado a estar cifrado.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**    
Esta vulnerabilidad podría provocar un funcionamiento incorrecto de la característica de seguridad. Para aprovechar este problema, es necesario que el atacante esté supervisando el tráfico de red. El atacante podrá leer y modificar el contenido del tráfico de red observado desde su posición estratégica concreta. En circunstancias normales, las directivas IPsec protegen este tráfico contra la intercepción mediante el cifrado del contenido. Un atacante no podría provocar que se produzca esta situación a menos que anteriormente haya obtenido permisos administrativos en el controlador de dominio. En vez de eso, debe esperar que un administrador, sin darse cuenta, haya configurado incorrectamente el conjunto de reglas IPsec y provoque que la información se transmita en texto sin cifrar.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**    
Las estaciones de trabajo de Windows Vista y los servidores de Windows Server 2008 sujetos a directivas IPsec que se encuentren en un dominio de Windows Server 2008 actualizados desde un dominio de Windows Server 2003 están más expuestos.
  
**¿Cómo funciona esta actualización?**    
La actualización elimina la vulnerabilidad al garantizar que las reglas IPsec se procesan de la manera adecuada.
  
**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**    
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable. En el momento de emitir este boletín de seguridad, Microsoft no había recibido información alguna que indicara que esta vulnerabilidad se hubiera divulgado.
  
**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba usando esta vulnerabilidad?**    
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.
