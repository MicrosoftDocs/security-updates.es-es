Security Bulletin

Boletín de seguridad de Microsoft MS08-030 - Crítica
====================================================

Una vulnerabilidad en la pila de Bluetooth podría permitir la ejecución remota de código (951376)
-------------------------------------------------------------------------------------------------

Publicado: martes, 10 de junio de 2008 | Actualizado: jueves, 19 de junio de 2008

**Versión:** 2.0

### Información general

#### Resumen ejecutivo

Esta actualización de seguridad resuelve una vulnerabilidad de la que se ha informado de forma privada en la pila de Bluetooth que podría permitir la ejecución remota de código. Un atacante que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario.

Esta actualización de seguridad se considera crítica para todas las ediciones compatibles de Windows XP y Windows Vista. Para obtener más información, consulte la subsección **Software afectado y no afectado**, en esta sección.

La actualización de seguridad corrige la vulnerabilidad al modificar la forma en que la pila de Bluetooth trata una gran cantidad de solicitudes de descripción de servicio. Para obtener más información acerca de la vulnerabilidad, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad**.

**Recomendación.** Microsoft recomienda a sus clientes que apliquen la actualización inmediatamente.

**Problemas conocidos.** Ninguna

Vulnerabilidad de Bluetooth (CVE-2008-1453)  
-------------------------------------------
  
<span></span>
Existe una vulnerabilidad de ejecución remota en la pila de Bluetooth en Microsoft Windows porque la pila de Bluetooth no trata correctamente una gran cantidad de solicitudes de descripción de servicio. La vulnerabilidad podría permitir a un atacante ejecutar código con privilegios elevados. Un atacante que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2008-1453](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1453).
  
#### Factores atenuantes para la vulnerabilidad de Bluetooth (CVE-2008-1453)
  
El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:
  
-   Esta vulnerabilidad sólo afecta sistemas con la capacidad Bluetooth.
  
#### Soluciones provisionales para la vulnerabilidad de Bluetooth (CVE-2008-1453)
  
La solución provisional hace referencia a un cambio de opción o configuración que no corrige la vulnerabilidad subyacente pero ayuda a bloquear los tipos de ataque conocidos antes de aplicar la actualización. Microsoft ha probado las soluciones provisionales y señala en la explicación si una solución provisional reduce la funcionalidad.
  
-   **Evitar que el equipo acepte nuevas conexiones Bluetooth**  
  
**Método interactivo**
  
1.  En el **Panel de control**, seleccione **Vista clásica**.  
2.  Seleccione **Dispositivos Bluetooth**.  
3.  Haga clic en la ficha **Opciones**.  
4.  Desactive la casilla **Permitir que los dispositivos Bluetooth encuentren este equipo**.  
5.  Haga clic en **Aceptar**.
  
**Cómo deshacer la solución provisional**
  
1.  En el **Panel de control**, seleccione **Vista clásica**.  
2.  Seleccione **Dispositivos Bluetooth**.  
3.  Haga clic en la ficha **Opciones**.  
4.  Active la casilla **Permitir que los dispositivos Bluetooth encuentren este equipo**.  
5.  Haga clic en **Aceptar**.
  
**Método de secuencia de comandos de PowerShell**
  
Ejecute los comandos siguientes como un administrador con privilegios elevados:
  
$BthDriverKeys = Get-ChildItem "HKLM:\\System\\CurrentControlSet\\Enum\\usb" -recurse -ErrorAction SilentlyContinue | where {$\_.GetValue("Service") -eq "BTHUSB"}  
foreach($Key in $BthDriverKeys) { Set-ItemProperty -Path $Key.OpenSubKey("Device Parameters").Name.Replace("HKEY\_LOCAL\_MACHINE", "HKLM:") -Name "Write Scan Enable" -Value 0 }
  
**Cómo deshacer la solución provisional**
  
Ejecute los comandos siguientes como un administrador con privilegios elevados:
  
$BthDriverKeys = Get-ChildItem "HKLM:\\System\\CurrentControlSet\\Enum\\usb" -recurse -ErrorAction SilentlyContinue | where {$\_.GetValue("Service") -eq "BTHUSB"}  
foreach($Key in $BthDriverKeys) { Set-ItemProperty -Path $Key.OpenSubKey("Device Parameters").Name.Replace("HKEY\_LOCAL\_MACHINE", "HKLM:") -Name "Write Scan Enable" -Value 3 }
  
-   **Desactivar y deshabilitar el controlador Bluetooth**  
  
También puede detener y deshabilitar el controlador si usa el siguiente comando en el símbolo del sistema (disponible en Windows XP y posterior, y en el kit de recursos de Windows 2000):
  
sc config bthport start= disabled
  
**Cómo deshacer la solución provisional**
  
Puede volver a habilitar el controlador Bluetooth si usa el siguiente comando en el símbolo del sistema (disponible en Windows XP y posterior, y en el kit de recursos de Windows 2000):
  
sc config bthport start= demand
  
#### Preguntas más frecuentes sobre la vulnerabilidad de Bluetooth (CVE-2008-1453)
  
**¿Cuál es el alcance de esta vulnerabilidad?**  
Existe una vulnerabilidad de ejecución remota de código en la pila de Bluetooth en Microsoft Windows. Un atacante que consiguiera aprovechar esta vulnerabilidad podría ejecutar código con privilegios elevados. A continuación, podría lograr el control completo de un sistema afectado.
  
**¿Cuál es la causa de esta vulnerabilidad?**  
La pila de Windows Bluetooth no trata correctamente una gran cantidad de solicitudes de descripción de servicio.
  
**¿Qué es Bluetooth?**  
Bluetooth es un protocolo estándar del sector que permite la conectividad inalámbrica de equipos, dispositivos de mano, teléfonos móviles y otros dispositivos. Vea el [artículo de Bluetooth de MSDN](http://msdn.microsoft.com/en-us/library/aa362932(vs.85).aspx) para obtener más información.
  
**¿Para qué puede usar un atacante esta vulnerabilidad?**  
Un atacante que aprovechara esta vulnerabilidad podría obtener el control completo del sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
Para aprovechar esta vulnerabilidad, un atacante tendría que enviar rápidamente una gran cantidad de paquetes SDP (protocolo de detección de servicios) especialmente diseñados a un sistema afectado. La forma en que el sistema reaccione a estos paquetes permitiría al atacante obtener control completo del sistema afectado.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Todos los sistemas con Bluetooth habilitado están expuestos a esta vulnerabilidad.
  
**¿Cómo funciona esta actualización?**  
La actualización elimina la vulnerabilidad al mejorar la forma en que la pila de Bluetooth trata una gran cantidad de paquetes SDP.
  
**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable.
  
**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba usando esta vulnerabilidad?**  
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.
