Security Bulletin

Boletín de seguridad de Microsoft MS10-031 - Crítica
====================================================

Una vulnerabilidad en Microsoft Visual Basic para Aplicaciones podría permitir la ejecución remota de código (978213)
---------------------------------------------------------------------------------------------------------------------

Publicado: martes, 11 de mayo de 2010 | Actualizado: miércoles, 19 de mayo de 2010

**Versión:** 1.1

### Información general

#### Resumen ejecutivo

Esta actualización de seguridad resuelve una vulnerabilidad de la que se ha informado de forma privada en Microsoft Visual Basic para Aplicaciones. La vulnerabilidad podría permitir la ejecución remota de código si una aplicación host abre y pasa un archivo especialmente diseñado al módulo de tiempo de ejecución de Visual Basic para Aplicaciones. Si un usuario inicia sesión con derechos de usuario administrativos, un intruso que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.

Esta actualización de seguridad se considera crítica para todas las versiones compatibles del SDK de Microsoft Visual Basic para Aplicaciones y las aplicaciones de terceros que usan Microsoft Visual Basic para Aplicaciones. Esta actualización de seguridad también se considera importante para todas las ediciones compatibles de Microsoft Office XP, Microsoft Office 2003 y 2007 Microsoft Office System. Para obtener más información, consulte la subsección **Software afectado y no afectado**, en esta sección.

La actualización corrige la vulnerabilidad al modificar la forma en que Visual Basic para Aplicaciones busca los controles ActiveX incrustados en los documentos. Para obtener más información acerca de la vulnerabilidad, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad**.

**Recomendación.** Microsoft recomienda a sus clientes que apliquen la actualización inmediatamente.

**Problemas conocidos.** En el [artículo 978213 de Microsoft Knowledge Base](http://support.microsoft.com/kb/978213) se señalan los problemas detectados que pueden experimentar los clientes al instalar esta actualización de seguridad. Se documentan asimismo las soluciones propuestas. Cuando los problemas conocidos y las soluciones recomendadas corresponden sólo a versiones específicas de este software, este artículo proporciona vínculos a más artículos.

#### Software afectado y no afectado

El software siguiente se ha probado para determinar las versiones o ediciones que están afectadas. Otras versiones o ediciones han llegado al final de su ciclo de vida o no están afectadas. Para determinar el ciclo de vida del soporte técnico de su versión o edición de software, visite [Ciclo de vida del soporte técnico de Microsoft](http://go.microsoft.com/fwlink/?linkid=21742).

Preguntas más frecuentes (P+F) relacionadas con esta actualización de seguridad  
-------------------------------------------------------------------------------
  
<span></span>  
**¿Por qué se ha revisado este boletín el 19 de mayo de 2010?**    
Microsoft ha revisado este boletín para indicar de forma más clara que las actualizaciones de Microsoft Office se aplican a todos los conjuntos de programas de Microsoft Office y otro software de Microsoft Office que contienen una versión vulnerabilidad de VBE6.dll. Se incluyen, entre otras, las versiones compatibles de Microsoft Office Visio y Microsoft Office Project. También se incluye el software de Office que contiene VBE6.dll pero no obtiene acceso al código vulnerable, como las ediciones compatibles de Microsoft Office Outlook y Microsoft Office OneNote.
  
Se trata únicamente de un cambio de boletín. No hubo cambios en la lógica de detección ni en los archivos de actualización. Los clientes que ya han actualizado correctamente sus sistemas, incluidos los clientes con la actualización automática habilitada, no necesitan realizar ninguna acción. Es posible que los clientes que no han instalado esta actualización anteriormente tengan que volver a determinar si sus sistemas requieren esta actualización según la revisión de la tabla Software afectado y la información proporcionada en esta pregunta más frecuente de actualización.
  
**La actualización es para un componente compartido que usa Microsoft Office. ¿La actualización se ofrecerá al software de Microsoft Office que incluye el componente compartido, incluso si el software no accede al código vulnerable?**    
Sí, la actualización se ofrecerá a los sistemas donde se detecte el componente de Office compartido vulnerable. Esto se aplica incluso en los casos en los que el software de Office incluya VBE6.dll, pero no obtenga acceso al código vulnerable. Consulte en la siguiente tabla los ejemplos de software de Office a los que se aplica esta actualización.

|---------------------------------------------------------------------------------------------|-----------------------------------------|-------------------------------------------------|  
| Microsoft Office XP, Microsoft Office 2003, 2007 Microsoft Office System                    | Sí                                      | Sí, componentes como Word, Excel y PowerPoint   |  
| Microsoft Office OneNote 2003, Microsoft Office OneNote 2007                                | Sí                                      | No                                              |  
| Microsoft Office Outlook 2003, Microsoft Office Outlook 2007                                | Sí                                      | No                                              |  
| Microsoft Office Project 2002, Microsoft Office Project 2003, Microsoft Office Project 2007 | Sí                                      | Sí                                              |

**Nota** Utilice esta tabla sólo como guía. Incluye una muestra del software que recibe más preguntas.
  
En los casos en los que se ofrece la actualización, Microsoft recomienda a los clientes que instalen la actualización independientemente de si el software obtiene o no acceso al código vulnerable. La instalación de la actualización ayudará a los clientes a asegurarse de que ejecutan la versión más actualizada de VBE6.dll que usa el software de Microsoft Office.
  
**¿La oferta para actualizar una versión no vulnerable constituye un problema en el sistema de actualización de Microsoft?**    
No, el mecanismo de actualización funciona correctamente ya que detecta una versión anterior de los archivos en el sistema de la que hay en el paquete de actualización y, por lo tanto, ofrece la actualización.
  
**¿Dónde están los detalles de información de archivos?**    
Consulte en las tablas de referencia de la sección **Implementación de la actualización de seguridad** la ubicación de los detalles de información de archivo.
  
**He aplicado las actualizaciones de seguridad de Microsoft necesarias, pero todavía tengo una versión afectada del módulo de tiempo de ejecución de Visual Basic para Aplicaciones (VBE6.dll) en mi sistema. ¿Cómo actualizo esta DLL?**    
Existen casos en los que el sistema puede seguir teniendo una versión afectada de VBE6.dll incluso después de haber instalado las actualizaciones de seguridad necesarias para Microsoft Office y la actualización para Microsoft Visual para Aplicaciones que se incluyen en este boletín.
  
Si una versión compatible de Microsoft Office ha instalado el archivo VBE6.dll en el sistema, al aplicar la actualización de seguridad para la versión afectada de Microsoft Office se reemplazará VBE6.dll con la versión que corrige la vulnerabilidad descrita en este boletín. No obstante, si una aplicación de terceros ha instalado VBE6.dll en el sistema, puede que tenga que instalar una actualización para dicho programa.
  
A fin de actualizar VBE6.dll para las aplicaciones de terceros, existen dos escenarios posibles según la implementación de VBA de la aplicación de terceros. Si sabe que la aplicación de terceros es compatible con los procedimientos recomendados para usar un componente compartido como un ensamblado paralelo, al aplicar la actualización de Microsoft Visual Basic para Aplicaciones (KB974945) se reemplazará VBE6.dll en la ubicación compartida con la versión actualizada que corrige la vulnerabilidad descrita en este boletín.
  
Por otro lado, si la aplicación de terceros no coloca VBE6.DLL en la ubicación compartida según los procedimientos recomendados, debe ponerse en contacto con el desarrollador de la aplicación de terceros y pedirle que le proporcione una versión actualizada de su aplicación que contenga una versión más reciente de VBE6.dll que corrija la vulnerabilidad descrita en este boletín. Tenga en cuenta que no todas las aplicaciones de terceros que instalan VBE6.dll están afectadas por la vulnerabilidad descrita en este boletín porque pueden no usar VBE6.dll para buscar controles ActiveX de un modo que resulte vulnerable. Sin embargo, sólo el desarrollador de esa aplicación puede tomar esa determinación.
  
**Esta actualización de seguridad sólo se aplica al software de Microsoft. ¿Cómo puedo detectar que las aplicaciones de terceros han implementado una versión afectada del módulo de tiempo de ejecución de Visual Basic para Aplicaciones (VBE6.dll) en mi sistema?**    
Las aplicaciones de terceros que admiten VBA podrían implementar VBE6.dll en una ubicación que no se actualice mediante esta actualización de seguridad. En el caso de que tenga una aplicación de terceros que haya incluido su propia copia de VBE6.dll para contribuir a garantizar que el sistema está completamente protegido frente a la vulnerabilidad descrita en este boletín, debe ponerse en contacto con el desarrollador o proveedor responsable para obtener soporte técnico para la aplicación de terceros directamente.
  
Para obtener detalles acerca de cómo buscar copias de VBE6.dll en su sistema, vea el [artículo 978213 de Microsoft Knowledge Base](http://support.microsoft.com/kb/978213). La búsqueda de todas las copias de VBE6.dll en su sistema puede ayudarle a identificar las copias potencialmente vulnerables que han instalado aplicaciones de terceros.
  
**Soy un desarrollador de aplicaciones de terceros y uso el módulo de tiempo de ejecución de Microsoft Visual Basic para Aplicaciones en mi aplicación. ¿Es mi aplicación vulnerable y cómo la actualizo?**    
Los desarrolladores que redistribuyen el módulo de tiempo de ejecución de Microsoft Visual Basic para Aplicaciones VBE6.dll deben asegurarse de que actualizan la versión del módulo de tiempo de ejecución de Microsoft Visual Basic para Aplicaciones instalado con su aplicación mediante la descarga de una versión actualizada del SDK de Microsoft Visual Basic para aplicaciones desde [Summit Software Company](http://www.summsoft.com/). Para obtener más información acerca de los procedimientos recomendados para el uso de componentes redistribuidos, vea el [artículo 835322 de Microsoft Knowledge Base](http://support.microsoft.com/kb/835322) y el artículo de MSDN acerca de [aplicaciones aisladas y ensamblados paralelos](http://msdn.microsoft.com/en-us/library/aa375193.aspx).
  
**Soy un ISV. ¿Dónde está la actualización del SDK de Microsoft Visual Basic para Aplicaciones?**    
La actualización está disponible en [Summit Software Company](http://www.summsoft.com/). Summit Software Company es un proveedor de productos de software de personalización de aplicaciones y servicios de asistencia de integración a proveedores de software independientes (ISV) y desarrolladores corporativos. En junio de 1996, Summit Software llegó a un acuerdo con Microsoft para vender Microsoft Visual Basic para Aplicaciones y la tecnología y servicios de valor agregado relacionados. Summit sigue vendiendo y ofreciendo soporte técnico a Microsoft VBA.
  
**¿Por qué esta actualización es crítica para Microsoft Visual Basic para Aplicaciones, pero sólo importante para Microsoft Office?**    
Microsoft Office XP y versiones posteriores tienen integrada una característica que solicita la intervención del usuario para abrir, guardar o cancelar antes de abrir un documento. Este factor atenuante reduce la vulnerabilidad de crítica a importante porque requiere más que la intervención del usuario para aprovecharla.
  
**Uso Microsoft Office 2007 Service Pack 1. ¿Se ha incluido alguna característica de seguridad adicional en esta actualización?**    
Sí, como parte del modelo de servicio para Microsoft Office 2007, cuando los usuarios de Microsoft Office 2007 Service Pack 1 instalan esta actualización, sus sistemas se actualizan a la funcionalidad de seguridad que se publicó inicialmente con Microsoft Office 2007 Service Pack 2. Todas las actualizaciones publicadas después del 24 de abril de 2009 para Microsoft Office 2007 incluirán estas características de seguridad, que se presentaron en Microsoft Office 2007 Service Pack 2. Hemos probado exhaustivamente esta actualización, pero al igual que sucede con todas las actualizaciones, se recomienda que los usuarios lleven a cabo pruebas adecuadas al entorno y la configuración de sus sistemas.

Vulnerabilidad de daños en la memoria de la pila de VBE6.dll (CVE-2010-0815)
----------------------------------------------------------------------------

<span></span>
Existe una vulnerabilidad de ejecución remota de código en la forma en que Microsoft Visual Basic para Aplicaciones busca los controles ActiveX. La vulnerabilidad podría permitir la ejecución remota de código si una aplicación host abre y pasa un archivo especialmente diseñado a Visual Basic para Aplicaciones. Si un usuario inicia sesión con derechos de usuario administrativos, un intruso que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.

Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2010-0815](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0815).

#### Factores atenuantes para la vulnerabilidad de daños en la memoria de la pila de VBE6.dll (CVE-2010-0815)

El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:

-   De forma predeterminada, 2007 Microsoft Office System avisa al usuario con una advertencia de seguridad antes de activar un control ActiveX integrado en un documento de Office. Los usuarios que deciden no habilitar este control están protegidos por esta configuración predeterminada. Para obtener más información, vea el artículo de Microsoft Office Online acerca de la [habilitación y deshabilitación de controles ActiveX en los documentos de Office](http://office.microsoft.com/en-us/help/ha100310671033.aspx).
-   Un intruso que aprovechara esta vulnerabilidad podría conseguir el mismo nivel de derechos de usuario que el usuario local. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.
-   En el caso de un ataque basado en web, el intruso podría alojar un sitio web que contuviera una página web para aprovechar esta vulnerabilidad. Además, los sitios web vulnerables y los sitios web que aceptan o alojan contenido o anuncios proporcionados por el usuario podrían incluir contenido especialmente diseñado que permita aprovechar esta vulnerabilidad. Sin embargo, el atacante no podría en ningún caso obligar a los usuarios a visitar estos sitios web. En su lugar, el atacante tendría que persuadir a los usuarios para que visitaran el sitio web; por lo general, convenciéndoles para que hagan clic en un vínculo de un mensaje de correo electrónico o de Instant Messenger que lleve a los usuarios al sitio web del atacante y para que abran el archivo de Office especialmente diseñado.
-   La vulnerabilidad no puede aprovecharse automáticamente mediante el correo electrónico. Un usuario debe abrir un archivo adjunto a un mensaje de correo electrónico para que un ataque tenga éxito.

#### Preguntas más frecuentes sobre la vulnerabilidad de daños en la memoria de la pila de VBE6.dll (CVE-2010-0815)

**¿Cuál es el alcance de esta vulnerabilidad?**  
Se trata de una vulnerabilidad de ejecución remota de código en Microsoft Visual Basic para Aplicaciones (VBA). Un atacante que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.

**¿Cuál es la causa de esta vulnerabilidad?**  
La vulnerabilidad se debe a la forma en que Visual Basic para Aplicaciones busca los controles ActiveX en un documento que admite VBA. Como resultado, es posible que una aplicación host, como Microsoft Office o una aplicación de terceros desarrollada para la programabilidad de Visual Basic, pase un documento especialmente diseñado con controles ActiveX incrustados a VBA, con lo que se crea una condición que podría permitir la ejecución de código arbitrario.

**¿Qué es Visual Basic para aplicaciones (VBA)?**  
Microsoft VBA es una tecnología de programación diseñada para programar aplicaciones empaquetadas de escritorio cliente e integrar dichas aplicaciones con los datos y sistemas existentes. Microsoft VBA se basa en el sistema de programación Microsoft Visual Basic. Los productos de Microsoft Office incluyen VBA y lo utilizan para llevar a cabo determinadas funciones. VBA también puede utilizarse para crear aplicaciones personalizadas basadas en una aplicación host existente.

**¿Para qué puede usar un atacante esta vulnerabilidad?**  
Un atacante que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Por tanto, los usuarios cuyas cuentas estén configuradas con pocos derechos de usuario en el sistema correrían un riesgo menor que aquellos que cuenten con derechos de usuario administrativos.

**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
Para que el ataque tenga éxito, un usuario tendría que abrir un documento especialmente diseñado que admita VBA. Este documento podría tratarse de cualquier tipo de documento que admita VBA como, por ejemplo, un documento de Word, una hoja de cálculo de Excel o una presentación de PowerPoint.

En el supuesto de un ataque por correo electrónico, un atacante podría aprovechar la vulnerabilidad enviando al usuario un archivo de Office especialmente diseñado y convenciéndole de que lo abra.

En el caso de un ataque a través de web, el atacante tendría que alojar un sitio web que contuviera un documento que admita VBA, como un archivo de Office, destinado a intentar aprovechar esta vulnerabilidad. Además, los sitios web vulnerables y los sitios web que aceptan o alojan contenido proporcionado por el usuario podrían incluir contenido especialmente diseñado que permita aprovechar esta vulnerabilidad. El atacante no podría obligar a los usuarios a visitar un sitio web especialmente diseñado. En su lugar, el atacante tendría que persuadirles para que visitaran el sitio web; por lo general, convenciéndoles para que hagan clic en un vínculo de un mensaje de correo electrónico o de Instant Messenger que lleve a los usuarios al sitio web del atacante y para que abran el archivo de Excel especialmente diseñado.

**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Las estaciones de trabajo y los servidores de Terminal Server donde se use Visual Basic para Aplicaciones o Microsoft Office son los sistemas más expuestos. Los servidores podrían correr un mayor riesgo si los administradores permiten a los usuarios iniciar sesión en los servidores y ejecutar programas. Sin embargo, las prácticas recomendadas desaconsejan permitir esto.

**¿Cómo funciona esta actualización?**  
La actualización corrige la vulnerabilidad al modificar la forma en que Microsoft Visual Basic para Aplicaciones busca los controles ActiveX incrustados en los documentos.

**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable.

**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba usando esta vulnerabilidad?**  
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes.
