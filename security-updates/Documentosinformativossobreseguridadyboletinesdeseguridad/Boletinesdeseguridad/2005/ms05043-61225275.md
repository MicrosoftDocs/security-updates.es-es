Security Bulletin

Boletín de seguridad de Microsoft MS05-043 - Crítica
====================================================

Una vulnerabilidad en el servicio de cola de impresión podría permitir la ejecución remota de código (896423)
-------------------------------------------------------------------------------------------------------------

Publicado: martes, 9 de agosto de 2005 | Actualizado: miércoles, 18 de abril de 2007

**Versión:** 1.2

#### Resumen

**Quién debería leer este documento:** Los clientes que utilizan Microsoft Windows

**Consecuencia de la vulnerabilidad:** Ejecución remota de código

**Gravedad máxima:** Crítica

**Recomendación:** Los clientes deberían aplicar esta actualización inmediatamente.

**Actualizaciones de seguridad reemplazadas:** Ninguna

**Advertencias:** En el [artículo 908506 de Microsoft Knowledge Base](http://support.microsoft.com/kb/908506) se señalan los problemas detectados que pueden experimentar los clientes al instalar esta actualización de seguridad. Se documentan asimismo las soluciones propuestas. Para obtener más información, vea el [artículo 908506 de Microsoft Knowledge Base](http://support.microsoft.com/kb/908506).

**Ubicaciones de descarga de la actualización de seguridad y productos probados:**

**Software afectado:**

-   Microsoft Windows 2000 Service Pack 4: [Descargar la actualización](http://www.microsoft.com/downloads/details.aspx?familyid=3dd3b530-7f43-4c18-8298-6e8797431a5d)

-   Microsoft Windows XP Professional x64 Edition
-   Microsoft Windows Server 2003 Service Pack 1
-   Microsoft Windows Server 2003 con SP1 para sistemas con Itanium

Por lo que se refiere a otras versiones, quizá ya no sean compatibles con las actualizaciones de seguridad o no se vean afectadas. ### Información general

Resumen ejecutivo
-----------------

<span></span>
**Resumen ejecutivo:**

Esta actualización resuelve una vulnerabilidad descubierta recientemente, de la que se ha informado de forma privada. Existe una vulnerabilidad en el servicio de cola de impresión que podría permitir la ejecución remota de código. La vulnerabilidad se documenta en este boletín, en el apartado "Detalles de la vulnerabilidad".

Un atacante que aprovechara esta vulnerabilidad podría lograr el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario.

Es recomendable que los clientes apliquen la actualización inmediatamente.

Detalles de la vulnerabilidad
-----------------------------

<span></span>
#### Vulnerabilidad en el servicio de cola de impresión (CAN-2005-1984):

Existe una vulnerabilidad de ejecución remota de código en el servicio de cola de impresión que podría permitir a un atacante que consiga aprovechar esta vulnerabilidad tomar el control completo del sistema afectado.

#### Factores atenuantes de la vulnerabilidad en el servicio de cola de impresión (CAN-2005-1984):

-   En Windows XP Service Pack 2 y Windows Server 2003, esta vulnerabilidad está limitada a usuarios autenticados. Adicionalmente, para que este problema pueda dar lugar a un ataque remoto en estas versiones de sistema operativo, un usuario local con los permisos adecuados tendría que compartir una impresora o intentar la conexión a una impresora compartida. Si ningún usuario con los permisos adecuados comparte una impresora o intenta la conexión a una impresora compartida, para poder aprovechar esta vulnerabilidad, un atacante deberá disponer de credenciales de inicio de sesión válidas, además de ser capaz de iniciar una sesión local.
-   En Windows XP Service Pack 2 y Windows Server 2003, este problema resultaría en la denegación de servicio. En Windows XP Service Pack 2 y Windows Server 2003, este problema no se puede aprovechar para la ejecución remota de código ni para la elevación de privilegios.
    En otras versiones de sistema operativo, los intentos de aprovechar esta vulnerabilidad muy probablemente darían como resultado la denegación de servicio. Sin embargo, podría darse la ejecución remota de código.
-   Si se siguen las prácticas recomendadas relativas al uso de servidores de seguridad y se implementan las configuraciones de servidores de seguridad predeterminadas estándar, puede contribuirse a proteger una red de los ataques que se originen fuera del ámbito de la empresa. Se recomienda que los sistemas conectados a Internet tengan expuesta la cantidad mínima de puertos.

#### Soluciones provisionales de la vulnerabilidad en el servicio de cola de impresión (CAN-2005-1984):

Microsoft ha probado las siguientes soluciones provisionales. Aunque estas soluciones provisionales no corrigen la vulnerabilidad subyacente, ayudan a bloquear los tipos de ataque conocidos. Cuando una solución provisional reduce la funcionalidad, se identifica en la siguiente sección.

-   **Deshabilite el servicio de cola de impresión**

    La deshabilitación del servicio de cola de impresión le ayudará a proteger el sistema afectado de los intentos de aprovechar esta vulnerabilidad. Para deshabilitar el servicio de cola de impresión, siga estos pasos:

    1.  Haga clic en **Inicio** y, a continuación, en **Panel de control** .También puede, seleccionar **Configuración** y, a continuación, hacer clic en **Panel de control**.
    2.  Haga doble clic en **Herramientas administrativas**.
    3.  Haga doble clic en **Servicios**.
    4.  Haga doble clic en **Cola de impresión**.
    5.  En la lista **Tipo de inicio**, haga clic en **Deshabilitado**.
    6.  Haga clic en **Detener** y, a continuación, en **Aceptar**.

    También puede detener o deshabilitar el servicio de cola de impresión usando el siguiente comando en el símbolo del sistema:

    `sc stop Spooler & sc config Spooler start= disabled`

    **Consecuencias de la solución provisional:** Si deshabilita el servicio de cola de impresión, no podrá imprimir de forma local o remota. Por lo tanto, recomendamos esta solución únicamente en sistemas que no hacen uso del proceso de impresión.

-   **En Windows 2000 Server Service Pack 4, elimine el servicio de cola de impresión de la clave del Registro NullSessionPipes:**
    Windows 2000 Server Service Pack 4 y Windows XP Service Pack 1 permiten conexiones anónimas al servicio afectado. Para evitar intentos de aprovechar esta vulnerabilidad por parte de atacantes anónimos, elimine el servicio de cola de impresión de la subclave NullSessionPipes. Esta solución provisional no evitará ataques de usuarios autenticados. Utilice esta solución únicamente si no puede deshabilitar el servicio de cola de impresión.

    **Nota** El uso incorrecto del Editor del Registro puede originar problemas graves que conlleven la reinstalación del sistema operativo. Microsoft no garantiza que los problemas derivados del uso incorrecto del Editor del Registro puedan resolverse. El uso del Editor del Registro es responsabilidad suya. Para obtener información acerca de cómo modificar el Registro, consulte el tema “Cambiar claves y valores” de la Ayuda del Editor del Registro (Regedit.exe) o los temas “Agregar y eliminar valores del Registro” y “Editar valores del Registro” de la Ayuda de Regedt32.exe.

    **Nota** Es conveniente realizar una copia de seguridad del Registro antes de modificarlo.

    1.  Haga clic en **Inicio**, en **Ejecutar**, escriba "**regedt32**" (sin las comillas) y haga clic en **Aceptar**.
    2.  En el Editor del Registro, ubique la siguiente clave del registro:

        **HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\LanmanServer\\Parameters\\NullSessionPipes**

    3.  Edite la clave del Registro y elimine el valor **SPOOLSS**.
    4.  Reinicie el sistema afectado después de llevar a cabo estas acciones.

    **Consecuencias de la solución provisional:** No se permitirán conexiones anónimas al servicio de cola de impresión. Éste es el comportamiento predeterminado de versiones de sistema operativo posteriores.

#### Preguntas más frecuentes sobre la vulnerabilidad en el servicio de cola de impresión (CAN-2005-1984):

**¿Cuál es el alcance de esta vulnerabilidad?**

Se trata de una vulnerabilidad de ejecución remota de código. Un atacante que aprovechara esta vulnerabilidad podría lograr de forma remota el control completo de un sistema afectado. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Sin embargo, lo más probable es que los intentos de aprovechar esta vulnerabilidad tuvieran como consecuencia la denegación de servicio.

**¿Cuál es la causa de esta vulnerabilidad?**
La existencia de un búfer sin comprobar en el servicio de cola de impresión.

**¿Qué es el servicio de cola de impresión?**
El servicio de cola de impresión, Spoolsv.exe, es un archivo ejecutable instalado como servicio. La cola se carga durante el inicio del sistema operativo y continúa ejecutándose hasta que se cierra el sistema. Este servicio administra el proceso de impresión, que incluye tareas como la recuperación de la ubicación del controlador de impresora correcto, la carga de dicho controlador, la colocación en la cola de un trabajo de impresión de las llamadas de función de alto nivel y la programación de trabajos de impresión. Una vez completadas las tareas de un trabajo de impresión particular, el servicio de cola de impresión transfiere el trabajo al enrutador de impresión. Para obtener más información acerca del servicio de cola de impresión, visite el siguiente [sitio Web](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/techref/d58ce7b9-49cf-4f5e-95e9-1ade005c13e0.mspx).

**¿Para qué puede utilizar un atacante esta vulnerabilidad?**
Un atacante que aprovechara con éxito esta vulnerabilidad para la ejecución remota de código podría obtener el control completo del sistema afectado. En Windows XP Service Pack 2 y Windows Server 2003, este problema resultaría en la denegación de servicio. En otras versiones de sistema operativo, los intentos de aprovechar esta vulnerabilidad seguramente resultarían en la denegación de servicio. Sin embargo, podría darse la ejecución remota de código.

**¿Quién podría aprovechar esta vulnerabilidad?**
En Windows 2000 y Windows XP Service Pack 1, cualquier usuario anónimo que pudiera entregar un mensaje especialmente diseñado al sistema afectado podría intentar aprovechar esta vulnerabilidad. En Windows XP Service Pack 2 y Windows Server 2003, esta vulnerabilidad está limitada a usuarios autenticados. Un atacante autenticado quizás podría iniciar una sesión local en un sistema e intentar aprovechar esta vulnerabilidad en todas las versiones de sistema operativo afectadas.

**¿Cómo podría aprovechar un atacante la vulnerabilidad?**
Un atacante podría intentar aprovechar la vulnerabilidad de forma remota creando un mensaje especialmente diseñado y enviándolo a un sistema afectado. El mensaje podría provocar que el sistema afectado ejecutara código en versiones de sistema operativo y configuraciones vulnerables a ataques remotos. De forma predeterminada, Windows 2000 y Windows XP Service Pack 1 son vulnerables a ataques remotos. En Windows XP SP2 o en Windows Server 2003 no se puede crear un ataque remoto a menos que un usuario que disponga de los permisos adecuados comparta una impresora o intente conectarse a una impresora compartida.
Para aprovechar de forma local esta vulnerabilidad en todas las versiones de sistema operativo, un atacante primero tendría que iniciar sesión en el sistema. Posteriormente, puede ejecutar una aplicación especialmente diseñada que aproveche la vulnerabilidad.

**¿Qué sistemas están más expuestos a esta vulnerabilidad?**
Windows 2000 y Windows XP Service Pack 1 están más expuestos a esta vulnerabilidad. El riesgo para los sistemas con Windows XP Service Pack 2 y Windows Server 2003 es menor debido a los factores atenuantes adicionales existentes en estas versiones de sistema operativo. Sin embargo, los sistemas configurados como servidores de impresión son especialmente vulnerables a este problema.

**¿Puede aprovecharse esta vulnerabilidad a través de Internet?**
Sí. Un atacante podría intentar aprovechar esta vulnerabilidad a través de Internet. Si se siguen las prácticas recomendadas y se implementan las configuraciones predeterminadas normales en los servidores de seguridad, puede protegerse mejor de los ataques que se realicen a través de Internet. Microsoft proporciona información sobre cómo puede ayudar a proteger su PC. Los usuarios finales pueden visitar el [sitio web Proteja su PC](http://go.microsoft.com/fwlink/?linkid=21169). Los profesionales de TI pueden visitar el [sitio web Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171) (en inglés).

**¿Cómo funciona esta actualización?**
Esta actualización elimina la vulnerabilidad al modificar el modo que utiliza el servicio de cola de impresión para validar la longitud de los mensajes antes de transferirlos al búfer asignado.

**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente responsable. En el momento de emitir este boletín de seguridad, Microsoft no había recibido información alguna que indicara que esta vulnerabilidad se hubiera divulgado.

**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba utilizando esta vulnerabilidad?**
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.

Microsoft Systems Management Server (SMS) ofrece una solución empresarial altamente configurable para la administración de las actualizaciones. Mediante SMS, los administradores pueden identificar los sistemas basados en Windows que necesitan actualizaciones de seguridad, así como realizar la implementación controlada de las actualizaciones en la empresa con una repercusión mínima para los usuarios finales. Para obtener más información acerca de cómo pueden utilizar SMS 2003 los administradores para implementar actualizaciones de seguridad, visite el [sitio web de administración de revisiones de seguridad de SMS 2003](http://go.microsoft.com/fwlink/?linkid=22939). Los usuarios de SMS 2.0 también pueden usar [Software Updates Service Feature Pack](http://go.microsoft.com/fwlink/?linkid=33340) como ayuda para la implementación de actualizaciones de seguridad. Para obtener más información acerca de SMS, visite el [sitio web de SMS](http://go.microsoft.com/fwlink/?linkid=21158).

**Nota** SMS utiliza las herramientas Microsoft Baseline Security Analyzer, Microsoft Office Detection Tool y Enterprise Update Scanning Tool para proporcionar un amplio soporte técnico en la detección e implementación de las actualizaciones indicadas en los boletines de seguridad. Puede que estas herramientas no detecten algunas actualizaciones de software. Los administradores pueden usar las prestaciones de inventario de SMS en estos casos para concretar qué actualizaciones se deben aplicar en cada sistema. Para obtener más información acerca de este procedimiento, visite el siguiente [sitio web](http://go.microsoft.com/fwlink/?linkid=33341), en inglés. Algunas actualizaciones de seguridad pueden requerir derechos administrativos y que se reinicie el sistema. Los administradores pueden usar la utilidad Elevated Rights Deployment Tool (disponible en [SMS 2003 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=33387) y en [SMS 2.0 Administration Feature Pack](http://go.microsoft.com/fwlink/?linkid=21161)) para instalar estas actualizaciones.
