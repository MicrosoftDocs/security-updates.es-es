Security Bulletin

Boletín de seguridad de Microsoft MS11-013 - Importante
=======================================================

Vulnerabilidades en Kerberos podrían permitir la elevación de privilegios (2496930)
-----------------------------------------------------------------------------------

Publicado: martes, 8 de febrero de 2011 | Actualizado: miércoles, 16 de marzo de 2011

**Versión:** 2.0

### Información general

#### Resumen ejecutivo

Esta actualización de seguridad resuelve una vulnerabilidad de la que se ha informado de forma privada y una vulnerabilidad de la que se ha informado de forma pública en Microsoft Windows. La más grave de estas vulnerabilidades podría permitir la elevación de privilegios si un atacante local y autenticado instala un servicio malintencionado en un equipo unido al dominio.

Esta actualización de seguridad se considera importante para todas las ediciones compatibles de Windows XP, Windows Server 2003, Windows 7 y Windows Server 2008 R2. Para obtener más información, consulte la subsección **Software afectado y no afectado** , en esta sección.

Esta actualización corrige las vulnerabilidades al impedir el uso de algoritmos de hash débiles tanto en Windows Kerberos como en Windows KDC, y al evitar que el cliente degrade el estándar de cifrado a DES para la comunicación Kerberos entre el cliente y el servidor. Para obtener más información acerca de las vulnerabilidades, consulte la subsección Preguntas más frecuentes (P+F) de la entrada de vulnerabilidad específica en la sección siguiente, **Información sobre la vulnerabilidad** .

**Recomendación.** La mayoría de los clientes tienen habilitada la actualización automática y no deben realizar ninguna acción porque esta actualización de seguridad se descargará e instalará automáticamente. Los clientes que no han habilitado la actualización automática deben buscar las actualizaciones e instalar esta actualización manualmente. Para obtener información sobre las opciones de configuración específicas de la actualización automática, vea el [artículo 294871 de Microsoft Knowledge Base](http://support.microsoft.com/kb/294871) .

Para administradores e instalaciones empresariales, o usuarios finales que deseen instalar esta actualización de seguridad manualmente, Microsoft recomienda que los clientes apliquen la actualización a la primera oportunidad con el software de administración de actualizaciones o busquen las actualizaciones con el servicio [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747) .

Vea también la sección **Herramientas y consejos para la detección e implementación** , más adelante en este boletín.

**Problemas conocidos.** Ninguna

Vulnerabilidad de suma de comprobación sin clave en Kerberos (CVE-2011-0043)  
----------------------------------------------------------------------------
  
<span></span>
Existe una vulnerabilidad de elevación de privilegios en las implementaciones de Kerberos. La vulnerabilidad se debe a que la implementación de Microsoft Kerberos admite un mecanismo de hash débil, que puede permitir que se falsifiquen determinados aspectos de un vale de servicio de Kerberos. Un usuario o un atacante malintencionado que aprovechara esta vulnerabilidad podría obtener un símbolo (token) con privilegios elevados en el sistema afectado.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2011-0043](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0043) .
  
#### Factores atenuantes para la vulnerabilidad de suma de comprobación sin clave en Kerberos (CVE-2011-0043)
  
El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:
  
-   Esta vulnerabilidad no se puede aprovechar en los dominios donde los controladores de dominio ejecutan Windows Server 2008 o Windows Server 2008 R2.  
-   Para aprovechar esta vulnerabilidad, un atacante debe de tener unas credenciales de inicio de sesión válidas y ser capaz de iniciar una sesión local. Los usuarios anónimos o los usuarios remotos no pueden aprovechar esta vulnerabilidad.
  
#### Soluciones provisionales para la vulnerabilidad de suma de comprobación sin clave en Kerberos (CVE-2011-0043)
  
Microsoft no ha identificado soluciones provisionales para esta vulnerabilidad.
  
#### Preguntas más frecuentes sobre la vulnerabilidad de suma de comprobación sin clave en Kerberos (CVE-2011-0043)
  
**¿Cuál es el alcance de esta vulnerabilidad?**  
Se trata de una vulnerabilidad de elevación de privilegios. Un atacante local y autenticado podría ejecutar un servicio malintencionado en un sistema que podría enviar un vale de servicio especialmente diseñado para obtener un símbolo (token) que le conceda mayores privilegios en dicho sistema.
  
**¿Cuál es la causa de esta vulnerabilidad?**  
La vulnerabilidad se debe a que la implementación de Microsoft Kerberos en Microsoft Windows admite mecanismos de hash débiles, como CRC32.
  
**¿Qué es Kerberos?**  
Kerberos es un protocolo que se usa para la autenticación mutua de usuarios y servicios en una red abierta y no protegida. Permite que los servicios identifiquen correctamente al usuario de un vale de Kerberos sin tener que autenticar al usuario en el servicio. Esta operación se lleva a cabo mediante el uso de claves secretas compartidas.
  
El protocolo Kerberos usa claves secretas compartidas para cifrar y señalizar las credenciales de los usuarios. Un usuario se autentica mediante un centro de distribución de claves Kerberos (KDC). Tras dicha autenticación, el usuario puede solicitar un vale de servicio para obtener acceso a un determinado servicio de la red. Este vale incluye la identidad cifrada autenticada del usuario.
  
**¿Para qué puede utilizar un atacante esta vulnerabilidad?**  
Un atacante que consiguiera aprovechar esta vulnerabilidad podría ejecutar código arbitrario en el contexto de seguridad del sistema local. De esta forma, un intruso podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con estos privilegios elevados.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
Un atacante local y autenticado con privilegios para instalar una aplicación Kerberos malintencionada con privilegios limitados podría aprovechar esta vulnerabilidad mediante la instalación de un servicio malintencionado en un equipo unido al dominio. A continuación, este servicio podría generar un vale de servicio Kerberos especialmente diseñado que permita al atacante obtener privilegios de nivel de sistema en el equipo local.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Los sistemas con más riesgo de sufrir un ataque son las estaciones de trabajo y los servidores de Servicios de Terminal Server. Los servidores podrían correr un mayor riesgo si algún usuario que no tenga los suficientes permisos administrativos tiene la capacidad de iniciar sesiones en el servidor y ejecutar programas. Sin embargo, las prácticas recomendadas desaconsejan permitir esto.
  
**¿Cómo funciona esta actualización?**  
Esta actualización corrige la vulnerabilidad al impedir el uso de algoritmos de hash débiles tanto en Windows Kerberos como en Windows KDC.
  
**En el momento de emitir este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
Sí. Esa vulnerabilidad ya se ha divulgado. Se le ha asignado el número de vulnerabilidad y exposición común [CVE-2011-0043](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0043) .
  
**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba utilizando esta vulnerabilidad?**  
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes y no tenía constancia de que se hubiera publicado código demostrativo.
  
Vulnerabilidad de suplantación en Kerberos (CVE-2011-0091)  
----------------------------------------------------------
  
<span></span>
Existe una vulnerabilidad de suplantación en las implementaciones de Kerberos en Windows 7 y Windows Server 2008 R2. La vulnerabilidad existe porque es posible degradar la autenticación de Kerberos para usar DES en vez de los estándares de cifrado predeterminados y más sólidos que se incluyen en Windows 7 y Windows Server 2008 R2.
  
Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, consulte [CVE-2011-0091](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-0091) .
  
#### Factores atenuantes para la vulnerabilidad de suplantación en Kerberos (CVE-2011-0091)
  
El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:
  
-   La vulnerabilidad sólo afecta a la autenticación Kerberos.
  
#### Soluciones provisionales para la vulnerabilidad de suplantación en Kerberos (CVE-2011-0091)
  
Microsoft no ha identificado soluciones provisionales para esta vulnerabilidad.
  
#### Preguntas más frecuentes sobre la vulnerabilidad de suplantación en Kerberos (CVE-2011-0091)
  
**¿Cuál es el alcance de esta vulnerabilidad?**  
Se trata de una vulnerabilidad de suplantación. Esta vulnerabilidad podría permitir a un atacante suplantar las credenciales de un usuario legítimo o falsificar todo el tráfico de Kerberos en una sesión atacada.
  
**¿Cuál es la causa de esta vulnerabilidad?**  
La vulnerabilidad se debe a que Windows no aplica correctamente los estándares de cifrado predeterminados y más sólidos incluidos en Windows 7 y Windows Server 2008 R2, por lo que es posible que un ataque de tipo "Man in the middle" aplique una degradación de la comunicación Kerberos entre un cliente y un servidor a un estándar de cifrado más débil que el negociado originalmente.
  
**¿Qué es Kerberos?**  
Kerberos es un protocolo que se usa para la autenticación mutua de usuarios y servicios en una red abierta y no protegida. Permite que los servicios identifiquen correctamente al usuario de un vale de Kerberos sin tener que autenticar al usuario en el servicio. Esta operación se lleva a cabo mediante el uso de claves secretas compartidas.
  
El protocolo Kerberos usa claves secretas compartidas para cifrar y señalizar las credenciales de los usuarios. Un usuario se autentica mediante un centro de distribución de claves Kerberos (KDC). Tras dicha autenticación, el usuario puede solicitar un vale de servicio para obtener acceso a un determinado servicio de la red. Este vale incluye la identidad cifrada autenticada del usuario.
  
**¿Para qué puede utilizar un atacante esta vulnerabilidad?**  
Un atacante podría degradar el estándar de cifrado predeterminado a DES. Esto le permitiría leer y falsificar todo el tráfico Kerberos en esa sesión. Un atacante podría usar esta capacidad para suplantar al usuario que se ha autenticado durante la sesión de Kerberos.
  
**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
En una situación de ataque de tipo "Man in the middle", un atacante tendría que estar situado entre el cliente destino y el servidor para interceptar y degradar el estándar de cifrado predeterminado a DES.
  
**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Los sistemas Windows 7 y Windows Server 2008 R2 que usen la autenticación Kerberos son vulnerables a este ataque de degradación.
  
**¿Cómo funciona esta actualización?**  
La actualización corrige la vulnerabilidad al impedir que un ataque del tipo "Man in the middle" degrade el estándar de cifrado a DES para la comunicación Kerberos entre el cliente y el servidor.
  
**En el momento de emitir este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
No. Microsoft recibió información acerca de esta vulnerabilidad a través de una fuente de divulgación de vulnerabilidades coordinada.
  
**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba utilizando esta vulnerabilidad?**  
No. En el momento de publicar este boletín de seguridad, Microsoft no había recibido ninguna información que indicara que esta vulnerabilidad se hubiera utilizado para atacar a clientes.
