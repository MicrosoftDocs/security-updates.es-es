Boletín de seguridad de Microsoft MS14-060 - Importante
=======================================================

Una vulnerabilidad en Windows OLE podría permitir la ejecución remota de código (3000869)
-----------------------------------------------------------------------------------------

Publicado: 14 de octubre de 2014

**Versión:** 1.0

Información general
-------------------

### Resumen ejecutivo

Esta actualización de seguridad crítica resuelve una vulnerabilidad de la que se ha informado de forma privada en Microsoft Windows. La vulnerabilidad podría permitir la ejecución remota de código si un usuario abre un archivo de Microsoft Office que contiene un objeto OLE especialmente diseñado. Un atacante que consiguiera aprovechar esta vulnerabilidad podría ejecutar código arbitrario en el contexto del usuario actual. Si el usuario actual inicia sesión con derechos de administrador, el atacante podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Los clientes cuyas cuentas estén configuradas con menos derechos de usuario en el sistema correrían un riesgo menor que los que cuenten con derechos de usuario administrativos.

Esta actualización de seguridad se considera importante para todas las versiones compatibles de Microsoft Windows excepto Windows Server 2003. Para obtener más información, vea la sección **Software afectado**.

La actualización de seguridad corrige la vulnerabilidad al modificar la manera en que los objetos OLE se activan en Windows. Para obtener más información acerca de la vulnerabilidad, vea la subsección **Preguntas más frecuentes (P+F)** de la vulnerabilidad específica más adelante en este boletín.

**Recomendación.** La mayoría de los clientes tiene habilitada la actualización automática y no debe realizar ninguna acción porque esta actualización de seguridad se descargará e instalará automáticamente. Para obtener información acerca de las opciones de configuración específicas en las actualizaciones automática, vea el [artículo 294871 de Microsoft Knowledge Base](https://support.microsoft.com/kb/294871). En el caso de los clientes que no tienen habilitadas las actualizaciones automáticas, se pueden seguir los pasos de [Activar o desactivar las actualizaciones automáticas](http://go.microsoft.com/fwlink/?linkid=398470) para habilitar las actualizaciones automáticas.

Para administradores e instalaciones empresariales o usuarios finales que deseen instalar esta actualización de seguridad manualmente (incluidos los clientes que no han habilitado las actualizaciones automáticas), Microsoft recomienda que los clientes apliquen la actualización lo antes posible con el software de administración de actualizaciones o busquen las actualizaciones con el servicio [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747). Las actualizaciones también están disponibles mediante los vínculos de descarga de la tabla **Software afectado**, más adelante en este boletín.

Para obtener orientación adicional, vea la sección **Herramientas y consejos para la detección e implementación** de este boletín.

Para obtener más información acerca de los Service Packs de estas versiones de software, vea [Directiva de compatibilidad del ciclo de vida de los Service Packs](http://go.microsoft.com/fwlink/?linkid=89213).

Los clientes sin un contrato Alliance, Premier o Authorized pueden ponerse en contacto con su oficina de ventas local de Microsoft. Para obtener información de contacto, visite el sitio web de [información en todo el mundo de Microsoft](http://go.microsoft.com/fwlink/?linkid=33329), seleccione el país en la lista para ver una lista de números de teléfono. Para obtener más información, vea [Preguntas más frecuentes de la directiva del ciclo de vida del soporte técnico de Microsoft](http://go.microsoft.com/fwlink/?linkid=169557).

Vulnerabilidad de ejecución remota de código en Windows OLE (CVE-2014-4114)
---------------------------------------------------------------------------

Existe una vulnerabilidad en Windows OLE que podría provocar la ejecución remota de código si un usuario abre un archivo que contiene un objeto OLE especialmente diseñado. Un intruso que aprovechara esta vulnerabilidad podría conseguir el mismo nivel de derechos de usuario que el usuario que ha iniciado sesión. Si el usuario actual inicia sesión con derechos de administrador, el atacante podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Los clientes cuyas cuentas estén configuradas con menos derechos de usuario en el sistema correrían un riesgo menor que los que cuenten con derechos de usuario administrativos.

Para ver esta vulnerabilidad como una entrada estándar en la lista de vulnerabilidades y exposiciones comunes, vea [CVE-2014-4114](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2014-4114). 

### Factores atenuantes

El factor atenuante hace referencia a una opción, configuración común o práctica recomendada general, existente en un estado predeterminado, que podría reducir la gravedad de una vulnerabilidad. Los siguientes factores atenuantes podrían ser útiles en su situación:

-   Un atacante que aprovechara esta vulnerabilidad podría conseguir el mismo nivel de derechos de usuario que el usuario actual. Los clientes cuyas cuentas estén configuradas con menos derechos de usuario en el sistema correrían un riesgo menor que los que cuenten con derechos de usuario administrativos.
-   En el caso de un ataque a través de web, el atacante podría hospedar una página web que contuviera un archivo de Office especialmente diseñado destinado a intentar aprovechar esta vulnerabilidad. Sin embargo, el atacante no podría en ningún caso obligar a los usuarios a visitar estos sitios web. Por lo tanto, tendría que atraerlos al sitio web; por lo general, convenciéndoles para que hagan clic en un vínculo de un mensaje de correo electrónico o de Instant Messenger que lleve a los usuarios al sitio web del atacante.
-   Los archivos de Internet y de otras ubicaciones potencialmente poco seguras pueden contener virus, gusanos u otros tipos de malware que podrían dañar su equipo. Para proteger su equipo, los archivos de estas ubicaciones potencialmente poco seguras se abren en Vista protegida. Mediante Vista protegida, puede leer un archivo y consultar su contenido a la vez que reduce los riesgos. Vista protegida está habilitada de forma predeterminada.
     

### Soluciones provisionales

Las soluciones provisionales hacen referencia a un cambio de opción o configuración que no corrige la vulnerabilidad subyacente pero ayuda a bloquear los tipos de ataque conocidos antes de aplicar la actualización. Microsoft ha probado las soluciones provisionales y señala en la explicación si una solución provisional reduce la funcionalidad: 

-   **Deshabilitar el servicio WebClient**

    Para deshabilitar el servicio WebClient en Windows Vista, Windows 7, Windows Server 2008 y Windows Server 2012, siga estos pasos:

    1.  Haga clic en **Inicio**, en **Ejecutar**, escriba **Services.msc** y, a continuación, haga clic en **Aceptar**.
    2.  Haga clic con el botón secundario del mouse en **WebClient** y seleccione **Propiedades**.
    3.  Cambie el **tipo de inicio** a **Deshabilitado**. Si el servicio está en ejecución, haga clic en **Detener**.
    4.  Haga clic en **Aceptar** y, a continuación, salga de la consola de administración Servicios.
    
    Para deshabilitar el servicio WebClient en Windows 8 y 8.1:

    1.  Presione la **tecla con el logotipo Windows + S** para abrir **Búsqueda**.
    2.  Escriba **Services.msc** y, a continuación, haga clic en **Aceptar**.
    3.  Haga clic con el botón secundario del mouse en **WebClient** y seleccione **Propiedades**.
    4.  Cambie el **tipo de inicio** a **Deshabilitado**. Si el servicio está en ejecución, haga clic en **Detener**.
    5.  Haga clic en **Aceptar** y, a continuación, salga de la consola de administración Servicios.

    **Consecuencias de la solución provisional.** Si se deshabilita el servicio WebClient, las solicitudes del Sistema distribuido de creación y control de versiones web (WebDAV) no se transmiten. Además, no se iniciarán los servicios que dependan explícitamente del servicio de cliente web y en el registro del sistema se almacenará un mensaje de error. Por ejemplo, los recursos compartidos de WebDAV no serán accesibles desde el equipo cliente.

    **Cómo deshacer la solución provisional.**

    Para habilitar el servicio WebClient en Windows Vista, Windows 7, Windows Server 2008 y Windows Server 2012, siga estos pasos:

    1.  Haga clic en **Inicio**, en **Ejecutar**, escriba **Services.msc** y, a continuación, haga clic en **Aceptar**.
    2.  Haga clic con el botón secundario del mouse en **WebClient** y seleccione **Propiedades**.
    3.  Cambie el **tipo de inicio** a **Manual**. Si el servicio no está en ejecución, haga clic en **Iniciar**.
    4.  Haga clic en **Aceptar** y, a continuación, salga de la consola de administración Servicios. 
         

    Para habilitar el servicio WebClient en Windows 8 y 8.1:

    1.  Presione la **tecla con el logotipo Windows + S** para abrir **Búsqueda**.
    2.  Escriba **Services.msc** y, a continuación, haga clic en **Aceptar**.
    3.  Haga clic con el botón secundario del mouse en **WebClient** y seleccione **Propiedades**.
    4.  Cambie el **tipo de inicio** a **Manual**. Si el servicio no está en ejecución, haga clic en **Iniciar**.
    5.  Haga clic en **Aceptar** y, a continuación, salga de la consola de administración Servicios.

     

-   **Bloquear los puertos TCP 139 y 445**

    Estos puertos TCP se pueden usar para iniciar una conexión con el componente afectado. Al bloquearlos en el firewall de la empresa, para el tráfico entrante y saliente, ayudará a proteger los sistemas situados detrás de dicho firewall de intentos de aprovechar esta vulnerabilidad. Le recomendamos bloquear toda la comunicación entrante no solicitada de Internet para ayudar a impedir ataques que puedan usar otros puertos. Para obtener más información acerca de los puertos, visite el siguiente sitio web: [Asignaciones de puertos TCP y UDP](http://go.microsoft.com/fwlink/?linkid=21312). Para obtener más información acerca de Firewall de Windows, visite el siguiente sitio web: [Cómo configurar Firewall de Windows en un único equipo](http://www.microsoft.com/technet/security/smallbusiness/prodtech/windowsxp/cfgfwall.mspx).

    **Consecuencias de la solución provisional.**

    Los puertos TCP 139 y 445 se usan para servicios adicionales, incluidos [Sistema de archivos de Internet comunes](http://technet.microsoft.com/library/cc939973.aspx) (CIFS), administración de DNS, servicios de servicio NetBT, sesiones de uso compartido de impresora y mucho más. La deshabilitación de estos puertos podría afectar a la funcionalidad de los servicios.

-   **Bloquear el lanzamiento de ejecutables mediante archivos de información de instalación**

    [Archivos de información de instalación](http://go.microsoft.com/fwlink/?linkid=157952) (extensión de archivo .INF) son scripts que admiten el lanzamiento de los ejecutables mediante el uso de un verbo de “instalación”. El Registro del sistema almacena una instrucción que contribuye a la ejecución del verbo de instalación especificado en los archivos .INF; al quitar la instrucción almacenada, se bloquearán los métodos de ataque del uso de archivos .INF para distribuir código malintencionado.

    **Nota** El uso incorrecto del Editor del Registro puede originar problemas graves que conlleven la reinstalación del sistema operativo. Microsoft no garantiza que los problemas derivados del uso incorrecto del Editor del Registro puedan resolverse. El uso del Editor del Registro es responsabilidad suya. Para obtener información sobre cómo editar el Registro, consulte el tema “Cambiar claves y valores” de la Ayuda del Editor del Registro (Regedit.exe) o los temas “Agregar y eliminar valores del Registro” y “Editar valores del Registro” de la Ayuda de Regedt32.exe.

    Instrucciones para editar el Registro y quitar el verbo “Install” en Windows Vista, Windows 7, Windows Server 2008 y Windows Server 2012:

    1.  Haga clic en **Inicio**, haga clic en **Ejecutar**, en el cuadro **Abrir** escriba **Regedit** y, a continuación, haga clic en **Aceptar**.
    2.  Busque y, a continuación, seleccione la siguiente clave del Registro:
        HKEY\_CLASSES\_ROOT\\inffile\\shell\\Install\\command
    3.  Realice una copia de seguridad del Registro antes de efectuar cambios; haga clic con el botón secundario del mouse en el comando del nombre de la clave del Registro y, a continuación, seleccione **Exportar**.
    4.  En el cuadro de diálogo Exportar archivo del Registro, escriba inffile\_install\_configuration\_backup.reg y haga clic en Guardar.
    5.  Haga clic con el botón secundario del mouse en el nombre de la clave del Registro **command** y seleccione **Eliminar**.
    6.  Haga clic en **Sí** para confirmar la eliminación.
    
    Instrucciones para editar el Registro y quitar el verbo “Install” en Windows 8 y Windows 8.1:

    1.  Presione la tecla con el logotipo Windows + S para abrir Búsqueda.
    2.  Escriba **Regedit** en el cuadro **Abrir** y, a continuación, haga clic en **Aceptar**.
    3.  Busque y, a continuación, seleccione la siguiente clave del Registro:
        HKEY\_CLASSES\_ROOT\\inffile\\shell\\Install\\command
    4.  Realice una copia de seguridad del Registro antes de efectuar cambios; haga clic con el botón secundario del mouse en el comando del nombre de la clave del Registro y, a continuación, seleccione **Exportar**.
    5.  En el cuadro de diálogo Exportar archivo del Registro, escriba inffile\_install\_configuration\_backup.reg y haga clic en **Guardar**.
    6.  Haga clic con el botón secundario del mouse en el nombre de la clave del Registro **command** y seleccione **Eliminar**.
    7.  Haga clic en **Sí** para confirmar la eliminación.

    **Consecuencias de la solución provisional.**

    Es posible que las aplicaciones que se basan en el uso de archivos .INF para ejecutar una aplicación de instalación no se ejecuten.

    **Cómo deshacer la solución provisional.**

    Importe la copia de seguridad del Registro que ha guardado anteriormente:

    1.  Busque **inffile\_install\_configuration\_backup.reg**, el archivo de copia de seguridad del Registro que ha exportado anteriormente.
    2.  Haga clic con el botón secundario del mouse en los datos exportados y seleccione **Combinar**.
    3.  Haga clic en **Sí** para confirmar la importación de los datos del Registro de copia de seguridad y, a continuación, haga clic en **Aceptar**.

### Preguntas más frecuentes

**¿Cuál es el alcance de esta vulnerabilidad?**  
Se trata de una vulnerabilidad de ejecución remota de código en el contexto del usuario actual.

**¿Cuál es la causa de esta vulnerabilidad?**  
Esta vulnerabilidad se produce cuando un usuario descarga, o recibe, y, a continuación, abre un archivo de Microsoft Office especialmente diseñado que contiene objetos OLE.

**¿Qué es OLE?**  
OLE (vinculación e incrustación de objetos) es una tecnología que permite a las aplicaciones compartir datos y funcionalidad, como la capacidad para crear y editar datos compuestos. Los datos compuestos son los que contienen información en varios formatos. Por ejemplo, un documento de Microsoft Word compuesto puede contener una hoja de cálculo de Microsoft Excel insertada (o un objeto OLE). Esta también permite la edición directa; en lugar de ejecutar una nueva aplicación cuando se activa un objeto OLE, el usuario ve un nuevo conjunto de elementos de menú en su aplicación existente. Para obtener más información acerca de OLE, vea [Documentos compuestos](http://msdn.microsoft.com/library/ms693383).

**¿Cuál es la diferencia entre los archivos de presentación y de presentación con diapositivas de Microsoft PowerPoint?**  
Los archivos de datos de presentación de Microsoft PowerPoint (extensiones .PPTX o .PPT) normalmente se abren en el modo de edición. Los archivos de datos de presentación con diapositivas de Microsoft PowerPoint (extensiones .PPSX o .PPS) normalmente se abren en el modo de edición. Los archivos de datos de presentación con diapositivas de PowerPoint se pueden compartir los usuarios que no vayan a editar los datos.

**¿Para qué puede utilizar un atacante esta vulnerabilidad?**  
Un atacante que consiguiera aprovechar esta vulnerabilidad podría ejecutar código arbitrario en el contexto del usuario actual. Si el usuario actual inicia sesión con derechos de administrador, el atacante podría instalar programas; ver, cambiar o eliminar datos; o crear cuentas nuevas con todos los derechos de usuario. Los clientes cuyas cuentas estén configuradas con menos derechos de usuario en el sistema correrían un riesgo menor que los que cuenten con derechos de usuario administrativos.

**¿Cómo podría aprovechar un atacante la vulnerabilidad?**  
La intervención del usuario es necesaria para aprovechar esta vulnerabilidad. Un usuario que haya iniciado la sesión localmente debe abrir un archivo adjunto a un mensaje de correo electrónico que contenga un objeto OLE especialmente diseñado para que un ataque a través del correo electrónico tenga éxito. Hay muchos tipos distintos de documentos adjuntos que pueden contener los objetos OLE afectados. Todos los tipos de archivos de Office, así como muchos archivos de terceros, pueden contener un objeto OLE malintencionado.

En el supuesto de un ataque por correo electrónico, un atacante podría aprovechar la vulnerabilidad si envía al usuario un archivo especialmente diseñado y lo convence de que lo abra.

En el caso de un ataque a través de web, el atacante tendría que hospedar un sitio web que contuviera un archivo de PowerPoint destinado a intentar aprovechar esta vulnerabilidad. Además, los sitios web vulnerables y los sitios web que aceptan u hospedan contenido proporcionado por el usuario podrían incluir contenido especialmente diseñado que permita aprovechar esta vulnerabilidad. El atacante no podría obligar a los usuarios a visitar un sitio web malintencionado. Por lo tanto, tendría que atraerlos al sitio web; para ello debería, por ejemplo, incitarles a hacer clic en un vínculo que les llevara al sitio web del atacante.

**¿Qué sistemas están más expuestos a esta vulnerabilidad?**  
Los servidores y los clientes de Microsoft Windows que abren archivos de datos de Microsoft Office que contienen objetos OLE están más expuestos.

**¿Cómo funciona esta actualización?**  
Esta actualización corrige las vulnerabilidades al modificar la forma en que los sistemas operativos afectados validan el uso de la memoria cuando se obtiene acceso a los objetos OLE.

**En el momento de publicar este boletín de seguridad, ¿se había divulgado ya esta vulnerabilidad?**  
No. Microsoft recibió información acerca de estas vulnerabilidades a través de una fuente de divulgación de vulnerabilidades coordinada.

**En el momento de publicar este boletín de seguridad, ¿había recibido Microsoft algún informe que indicase que se estaba utilizando esta vulnerabilidad?**  
Sí. Tenemos constancia de ataques dirigidos limitados en los que intervienen archivos de Microsoft PowerPoint 2007. Tantos los archivos de datos de presentación de Microsoft PowerPoint (extensiones de archivo .PPTX o .PPT) como los de presentación con diapositivas de PowerPoint (extensiones de archivo .PPSX o .PPS) se podrían usar en un escenario de ataque; si los archivos de presentación con diapositivas de PowerPoint se usan en un escenario de ataque, se podría instalar código arbitrario sin las preguntas de seguridad de [Control de cuentas de usuario](http://technet.microsoft.com/library/cc709691(v=ws.10).aspx) (UAC).

Para obtener información acerca de estas y otras herramientas que hay disponibles, vea [Herramientas de seguridad para profesionales de TI](http://technet.microsoft.com/security/cc297183).
